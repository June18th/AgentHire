# 一、相关技术栈
求职派的技术栈主流且相对聚焦，主打一个核心突出（SpringAI + 大模型），从大的角度出发，对应的技术栈如下

| 维度 | 选型 | 说明 |
| --- | --- | --- |
| 后端框架 | Spring Boot 4 + Java 21 | 使用 Java 后端熟悉的技术栈进行开发 |
| AI 接入 | Spring AI 2.0.0-M4 | 实现大模型的交互管理 |
| 智能体开发 | LangGraph4j | 智能体开发框架 |
| 数据层 | JPA + H2/MySQL | H2 快速本地跑，MySQL 一键切生产 |
| 前端 | React + Next.js 15 | SSR/SSG 兼顾 SEO 与体验 |

## 3.业务逻辑层
服务不能裸奔，所以必须有一层权限管理，或者叫网关（或者叫前置校验层）进行全局流量的统一管理

+ 在网关层，我们重点突出一下鉴权和上下文的维护方式（如果有限流、防刷、降级的当然也可以体现）
+ 鉴权策略：
    - http的api端点请求，基于JWT做身份认证鉴权
    - mcp的请求，基于HTTP Authorization进行身份认证鉴权
+ 上下文：
    - 由于是单体应用，因此我们直接使用 `TransmittableThreadLocal`，对于分布式架构，推荐基于redis做上下文管理


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753775975029-48ac3f92-ceed-45dc-9398-3159fbadb713.png)
