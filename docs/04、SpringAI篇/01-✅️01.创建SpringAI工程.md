### <font style="color:rgb(31, 35, 40);">1. 配置账号信息</font>
<font style="color:rgb(31, 35, 40);">上面创建的是一个 OpenAI(Chatgpt) 的大模型访问，需要我们提前备好梯子和密钥，并且进行账号充值才能使用，需要在</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">applicatin.properties</font>`<font style="color:rgb(31, 35, 40);">文件中添加密钥</font>

<font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">spring.ai.openai.api-key=<your-openai-api-key></font>

<font style="color:rgb(89, 99, 110);">出于安全考虑，可以通过配置环境变量的方式来配置密钥，如下</font>

```plain
# In application.yml
spring:
  ai:
    openai:
      api-key: ${OPENAI_API_KEY}
```

<font style="color:rgb(31, 35, 40);">然后设置环境变量</font>

```plain
# In your environment or .env file
export OPENAI_API_KEY=<your-openai-api-key>
```

### <font style="color:rgb(31, 35, 40);">3. 密钥注入方式</font>
<font style="color:rgb(31, 35, 40);">密钥除了直接在配置文件中写死之外，还可以通过前面介绍的环境变量方式来注入，除此之外，再介绍一种本地开发时命令行参数注入的方式</font>

<font style="color:rgb(31, 35, 40);">为了避免</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">api-key</font>`<font style="color:rgb(31, 35, 40);">的泄露，可以借助SpEL的方式来注入，通过在启动时，添加参数来设置apiKey：</font>

+ <font style="color:rgb(31, 35, 40);">唤起编辑启动命令行窗口</font>
+ <font style="color:rgb(31, 35, 40);">点击</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Modify options</font>`
+ <font style="color:rgb(31, 35, 40);">在弹窗中选中 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Program arguments</font>`<font style="color:rgb(31, 35, 40);">，然后会看到启动类右边新增了一个输入框</font>
+ <font style="color:rgb(31, 35, 40);">在输入框中添加</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">--spring.ai.zhipuai.api-key=<your-zhipuai-api-key></font>`

**<font style="color:rgb(31, 35, 40);">说明：为什么不建议直接在配置文件中写密钥？</font>**

+ <font style="color:rgb(31, 35, 40);">因为非常容易泄露，本地测试时，最简单的当然是直接在配置文件中写密钥，但是当我们将项目推送到git时，很容易忘了将密钥删除，这样，密钥就泄露了（即便你后续发现了再删除，但是git的历史提交中依然能找到你的密钥，你就只能删除它了）</font>

## <font style="color:rgb(31, 35, 40);">三、项目运行</font>
<font style="color:rgb(31, 35, 40);">初始化完成之后，我们来写一个简单的Chat服务，体验一下如何和大模型进行交互。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">新建一个ChatController类，用于接收用户的提问，并返回模型生成的答案</font>

```java
@RestController
public class ChatController {

    private final ZhiPuAiChatModel chatModel;

    @Autowired
    public ChatController(ZhiPuAiChatModel chatModel) {
        this.chatModel = chatModel;
    }

    @GetMapping("/ai/generate")
    public Map generate(@RequestParam(value = "message", defaultValue = "Tell me a joke") String message) {
        return Map.of("generation", chatModel.call(message));
    }

    @GetMapping("/ai/generateStream")
    public Flux<ChatResponse> generateStream(@RequestParam(value = "message", defaultValue = "Tell me a joke") String message) {
        var prompt = new Prompt(new UserMessage(message));
        return chatModel.stream(prompt);
    }
}
```

<font style="color:rgb(31, 35, 40);">上面分别提供同步/流式调用大模型两种方式，测试时，你可以选择任意一种方式</font>
