<font style="color:rgb(31, 35, 40);">通常情况下，如果我们不显式要求大模型返回什么样的数据结构时，大模型返回的数据结构往往是混乱的；这对于上层的业务开发来说，如何将大模型返回的关键信息映射为结构化的数据模型是一个非常困难的事情。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">SpringAI提供了一系列转换器来解决上面的痛点；接下来我们来具体看一下，怎么处理返回结果。</font>

## <font style="color:rgb(31, 35, 40);">一、实例演示</font>
<font style="color:rgb(31, 35, 40);">首先我们需要创建一个SpringAI的项目，基本流程同</font><font style="color:rgb(31, 35, 40);"> </font>[<font style="color:rgb(9, 105, 218);">创建一个SpringAI-Demo工程</font>](https://github.com/liuyueyi/spring-ai-demo/blob/master/docs/01.%E5%88%9B%E5%BB%BA%E4%B8%80%E4%B8%AASpringAI-Demo%E5%B7%A5%E7%A8%8B.md)

### <font style="color:rgb(31, 35, 40);">1. 初始化</font>
<font style="color:rgb(31, 35, 40);">创建一个MVC的API，用于和大模型交互。</font>

```java
@RestController
public class ChatController {

    public ChatController(ZhiPuAiChatModel chatModel) {
```

### <font style="color:rgb(31, 35, 40);">2. BeanOutputConverter</font>
<font style="color:rgb(31, 35, 40);">借助</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">BeanOutputConverter</font>`<font style="color:rgb(31, 35, 40);">可以将返回结果映射为java的POJO类，首先需要我们定义一个提示词模板，比如说查询某个导演的作品</font>

```plain
帮我返回五个{actor}导演的电影名，要求中文返回
```

<font style="color:rgb(31, 35, 40);">我们希望返回的结构如下:</font>

```plain
{
  "actor": "周星驰",
  "movies": [
    "大话西游之大圣娶亲",
    "喜剧之王",
    "功夫",
    "西游降魔篇",
    "长江七号"
  ]
}
```

<font style="color:rgb(31, 35, 40);">因此我们可以定义一个record，承接返回的结果</font>

```java
record ActorsFilms(String actor, List<String> movies) {
}
```

<font style="color:rgb(31, 35, 40);">借助</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">来实现结果解析</font>

```java
/**
 * 基于ChatClient实现返回结果的结构化映射
 *
 * @param actor
 * @return
 */
@GetMapping("/ai/generate")
public ActorsFilms generate(@RequestParam(value = "actor", defaultValue = "周星驰") String actor) {
    PromptTemplate template = new PromptTemplate("帮我返回五个{actor}导演的电影名，要求中文返回");
    Prompt prompt = template.create(Map.of("actor", actor));

    ActorsFilms films = ChatClient.create(chatModel).prompt(prompt).call().entity(ActorsFilms.class);
    return films;
}
```

<font style="color:rgb(31, 35, 40);">为什么上面的方式就可以实现结果映射为java的POJO类？</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);"> 我们可以debug一下</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient.create(chatModel).prompt(prompt).call()</font>`<font style="color:rgb(31, 35, 40);">返回的对象。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">从下面的截图中可以看到，在传递给大模型的请求中，context参数中，指定了要求大模型返回的数据格式（这里基于Advisors来实现的上下文数据附加/扩充提示词的功能）</font>

<font style="color:rgb(31, 35, 40);">当然也可以显示使用</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">BeanOutputConverter</font>`<font style="color:rgb(31, 35, 40);">基于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatModel</font>`<font style="color:rgb(31, 35, 40);">来实现</font>

```java
/**
 * 基于BeanOutputConverter实现返回结果结构化映射
 *
 * @param actor
 * @return
 */
@GetMapping("/ai/gen2")
public ActorsFilms gen2(@RequestParam(value = "actor", defaultValue = "周星驰") String actor) {
    BeanOutputConverter<ActorsFilms> beanOutputConverter = new BeanOutputConverter<>(ActorsFilms.class);
    String format = beanOutputConverter.getFormat();

    PromptTemplate template = new PromptTemplate("""
                                                 帮我返回五个{actor}导演的电影名
                                                 {format}
                                                 """);
    Prompt prompt = template.create(Map.of("actor", actor, "format", format));
    Generation generation = chatModel.call(prompt).getResult();
    if (generation == null) {
        return null;
    }
    return beanOutputConverter.convert(generation.getOutput().getText());
}
```

<font style="color:rgb(31, 35, 40);">注意上面的实现，在提示词模板中，新增了 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">{format}</font>`<font style="color:rgb(31, 35, 40);">，其值由 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">BeanOutputConverter</font>`<font style="color:rgb(31, 35, 40);"> 的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">getFormat()</font>`<font style="color:rgb(31, 35, 40);"> 方法获取；其实现原理是直接在提示词中添加了结构化的返回结果格式，因此，大模型返回的数据结构，会按照这个格式进行解析。</font>

### <font style="color:rgb(31, 35, 40);">3. 属性排序</font>
<font style="color:rgb(31, 35, 40);">借助</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">@JsonPropertyOrder</font>`<font style="color:rgb(31, 35, 40);">来实现排序，这个注解适用于record和普通的class</font>

```java
@JsonPropertyOrder({"actor", "movies"})
record ActorsFilms(String actor, List<String> movies) {
}

@GetMapping("/ai/genList")
public List<ActorsFilms> genList(@RequestParam(value = "actor1", defaultValue = "周星驰") String actor1,
                                 @RequestParam(value = "actor2", defaultValue = "张艺谋") String actor2) {
    List<ActorsFilms> actorsFilms = ChatClient.create(chatModel).prompt()
    .user(u ->
          u.text("帮我返回五个{actor1}和{actor2}导演的电影名，要求中文返回")
          .params(Map.of("actor1", actor1, "actor2", actor2)))
    .call()
    .entity(new ParameterizedTypeReference<List<ActorsFilms>>() {
    });
    return actorsFilms;
}
```

### <font style="color:rgb(31, 35, 40);">4. MapOutputConverter</font>
<font style="color:rgb(31, 35, 40);">上面介绍的是返回一个POJO，接下来看一下如何基于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">MapOutputConverter</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">来</font><font style="color:rgb(31, 35, 40);">实现map接收返回结果。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">这里使用的是 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ParameterizedTypeReference</font>`<font style="color:rgb(31, 35, 40);"> 来指定返回结果的类型</font>

<font style="color:rgb(31, 35, 40);">基于</font><font style="color:rgb(31, 35, 40);"> </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);"> </font><font style="color:rgb(31, 35, 40);">的方式</font>

```java
@GetMapping("/ai/genMap")
public Map genMap(@RequestParam(value = "actor", defaultValue = "周星驰") String actor) {
    Map<String, Object> actorsFilms = ChatClient.create(chatModel).prompt()
    .user(u ->
          u.text("帮我返回五个{actor}导演的电影名，要求中文返回")
          .param("actor", actor))
    .call()
    .entity(new ParameterizedTypeReference<Map<String, Object>>() {
    });
    return actorsFilms;
}
```

<font style="color:rgb(31, 35, 40);">基于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">MapOutputConverter</font>`<font style="color:rgb(31, 35, 40);">结合</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatModel</font>`<font style="color:rgb(31, 35, 40);">来实现</font>

```java
@GetMapping("/ai/genMap2")
public Map genMap2(@RequestParam(value = "actor", defaultValue = "周星驰") String actor) {
    MapOutputConverter mapOutputConverter = new MapOutputConverter();

    String format = mapOutputConverter.getFormat();
    PromptTemplate template = new PromptTemplate("""
                                                 帮我返回五个{actor}导演的电影名，要求中文返回
                                                 {format}
                                                 """);
    Prompt prompt = template.create(Map.of("actor", actor, "format", format));
    Generation generation = chatModel.call(prompt).getResult();
    Map<String, Object> result = mapOutputConverter.convert(generation.getOutput().getText());
    return result;
}
```

<font style="color:rgb(31, 35, 40);">两个接口的返回结果如下图，虽然都是返回的Map，但是仔细看之后，会发现他们的层级并不一样，基于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">返回的层级会多一层，返回的电影被放在了</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">movie</font>`<font style="color:rgb(31, 35, 40);">属性下，以列表的方式组织；而基于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">MapOutputConverter</font>`<font style="color:rgb(31, 35, 40);">返回的就是一层的map，key为数字；至于孰优孰劣，看个人喜好了。</font>

## <font style="color:rgb(31, 35, 40);">二、小结</font>
<font style="color:rgb(31, 35, 40);">本文主要介绍在SpringAI中如何结构化的处理大模型返回的结果，从使用方式来看，区分</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">和</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatModel</font>`<font style="color:rgb(31, 35, 40);">两种不同的使用姿势；其中前者更简单</font>

```java
// 方式一：ChatClient通过借助Advisor从上下文获取信息注入到提示词
ChatClient.create(chatModel).prompt()
.user("提示词")
.call()
.entity(xxx返回的对象类型);


// 方式二：在提示词中明确注入返回对象的格式
BeanOutputConverter<ActorsFilms> beanOutputConverter = new BeanOutputConverter<>(ActorsFilms.class);

String format = this.beanOutputConverter.getFormat();
String template = """
这里是提示词.
{format}
""";

Generation generation = chatModel.call(new PromptTemplate(this.template, Map.of("format", this.format)).create()).getResult();
ActorsFilms actorsFilms = this.beanOutputConverter.convert(this.generation.getOutput().getText());
```

<font style="color:rgb(31, 35, 40);">此外，发挥作用的核心是</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Converter</font>`<font style="color:rgb(31, 35, 40);">，SpringAI官方提供了下面这些具体的实现，基本上可以覆盖90%以上的业务场景； 若覆盖补全，则考虑通过自定义Converter来实现</font>
