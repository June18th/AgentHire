<font style="color:rgb(31, 35, 40);">SpringAI中，</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatModel</font>`<font style="color:rgb(31, 35, 40);">作为与大模型交互的具体实现，更上一层则推荐使用</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">，特别是在结构化输出、多轮对话的场景中，</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">提供了更方便的调用方式。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">如结构化输出，两者的写法对比如下</font>

```java
// 结构化返回场景:
// chatModel方式
BeanOutputConverter<ActorsFilms> beanOutputConverter = new BeanOutputConverter<>(ActorsFilms.class);
String format = beanOutputConverter.getFormat();

PromptTemplate template = new PromptTemplate("""
        帮我返回五个{actor}导演的电影名
        {format}
    """);
Prompt prompt = template.create(Map.of("actor", actor, "format", format));
Generation generation = chatModel.call(prompt).getResult();
if (generation == null) {
return null;
}
return beanOutputConverter.convert(generation.getOutput().getText());


// ChatClient方式
PromptTemplate template = new PromptTemplate("帮我返回五个{actor}导演的电影名，要求中文返回");
Prompt prompt = template.create(Map.of("actor", actor));
ActorsFilms films = ChatClient.create(chatModel).prompt(prompt).call().entity(ActorsFilms.class);
```

## <font style="color:rgb(31, 35, 40);">一、ChatClient基本使用</font>
### <font style="color:rgb(31, 35, 40);">1. 创建ChatClient</font>


<font style="color:rgb(31, 35, 40);">如果你的项目中只有一个大模型在用，并且是通过官方提供的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">starter</font>`<font style="color:rgb(31, 35, 40);">接入的，那可以直接使用SpringBoot自动装配的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient.Builder</font>`<font style="color:rgb(31, 35, 40);">来创建</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">，</font><font style="color:rgb(31, 35, 40);">如下。</font>

```java
@RestController
public class ChatController {
    private final ChatClient chatClient;

    public ChatController(ChatClient.Builder builder) {
        chatClient = builder.build();
    }
}
```

<font style="color:rgb(31, 35, 40);">但是，当一个应用中需要使用多个大模型时，就不能使用上面这种方式了，因为不知道到底用的是哪个模型，此时更建议使用</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatModel</font>`<font style="color:rgb(31, 35, 40);">进行创建。</font>

<font style="color:rgb(31, 35, 40);"></font>

```java
@RestController
public class ChatController {
    private final ChatClient chatClient;

    public ChatController(ChatModel chatModel) {
        chatClient = ChatClient.builder(chatModel).build();
    }
}
```

### <font style="color:rgb(31, 35, 40);">2. OpenAI兼容的API初始化方式</font>
<font style="color:rgb(31, 35, 40);">借助 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">OpenAiApi</font>`<font style="color:rgb(31, 35, 40);"> 与 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">OpenAiChatModel</font>`<font style="color:rgb(31, 35, 40);"> 类提供的 mutate() 方法，可以实现兼容的 OpenAI API 调用。</font>

```java
@Service
public class MultiModelService {

    private static final Logger logger = LoggerFactory.getLogger(MultiModelService.class);

    private ChatClient groqClient;

    public MultiModelService(OpenAiChatModel baseChatModel, OpenAiApi baseOpenAiApi) {
        try {
            // Derive a new OpenAiApi for Groq (Llama3)
            OpenAiApi groqApi = baseOpenAiApi.mutate()
            .baseUrl("https://api.groq.com/openai")
            .apiKey(System.getenv("GROQ_API_KEY"))
            .build();

            // Derive a new OpenAiChatModel for Groq
            OpenAiChatModel groqModel = baseChatModel.mutate()
            .openAiApi(groqApi)
            .defaultOptions(OpenAiChatOptions.builder().model("llama3-70b-8192").temperature(0.5).build())
            .build();

            groqClient = ChatClient.builder(groqModel).build();
        } catch (Exception e) {
        }
    }
}
```

### <font style="color:rgb(31, 35, 40);">3. 传入提示词</font>
在创建 ChatClient 时，我们需要传入一个 Prompt 对象来和大模型交互。Spring AI 提供了三种常见的用法：

#### **①、直接传字符串**
这是最简单的方式，把要问的内容直接丢进去就行：

```java
chatClient.prompt("为我写首诗").call().content();
```

这种写法表示用户输入的文本会直接作为消息传递给大模型。

#### **②、传入 Prompt 对象**
如果你想自己更细致地控制交互，可以直接 new 一个 Prompt 对象，把消息封装在里面：

```java
chatClient.prompt(new Prompt(new UserMessage("为我写首诗")))
.call()
.content();
```

这样做的好处是，交互的信息结构更清晰，可以自己来管控。

#### **③、Fluent 风格**
Spring AI 还支持 Fluent API，无参启动后可以一步步往里加系统消息、用户消息等提示词：

```java
chatClient.prompt()
    .system("你现在扮演盛唐著名的诗人李白，接下来我们进行对话")
    .user("为我写首诗")
    .call().content();
```

### <font style="color:rgb(31, 35, 40);">4. 响应ChatResponse</font>
<font style="color:rgb(31, 35, 40);">AI 模型返回的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatResponse</font>`<font style="color:rgb(31, 35, 40);"> 对象，封装了 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Generation</font>`<font style="color:rgb(31, 35, 40);"> 对象，以及一些元数据、token统计。</font>

```java
ChatResponse response = chatClient.prompt()
    .system("你现在扮演盛唐著名的诗人李白，接下来我们进行对话")
    .user("为我写首诗")
    .call().chatResponse();
```

<font style="color:rgb(31, 35, 40);">如果我们希望获取用户的token情况，可以在元数据中获取 </font><font style="color:rgb(31, 35, 40);background-color:rgb(246, 248, 250);">Usage usage = response.getMetadata().getUsage();</font>

<font style="color:rgb(31, 35, 40);"></font>

```java
// ChatResponse中实际以数组的方式承载 Generation 以应对多响应的场景
// 对于大部分场景，只需要获取第一个即可
Generation generation = response.getResult();
```

<font style="color:rgb(31, 35, 40);">如果需要将返回的String映射为实体类，可以考虑使用 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">entity()</font>`<font style="color:rgb(31, 35, 40);"> 来实现</font>

### <font style="color:rgb(31, 35, 40);">5. Flux流式调用</font>
`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">call</font>`<font style="color:rgb(31, 35, 40);">方法用来同步请求大模型，但更常见的方式是流式交互，问一个问题，模型一点一点的返回结果。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">对于</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChtClient</font>`<font style="color:rgb(31, 35, 40);">而言，要想实现流式调用，需要借助</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">stream()</font>`<font style="color:rgb(31, 35, 40);">方法，如</font>

@GetMapping(path = "/ai/fluxGen", produces = "text/event-stream")

            .system("你现在扮演盛唐著名的诗人李白，接下来我们进行对话")

## <font style="color:rgb(31, 35, 40);">二、ChatClient进阶使用</font>
### <font style="color:rgb(31, 35, 40);">1. 提示词模板替换</font>
`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);"> 的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Fluent</font>`<font style="color:rgb(31, 35, 40);"> API 支持含变量的用户/系统消息模板，运行时可进行替换。</font>

```java
@GetMapping("/ai/template")
public String template(@RequestParam(value = "role", defaultValue = "李白") String role,
                       @RequestParam(value = "msg", defaultValue = "你好") String msg) {
    return chatClient.prompt()
            .system(u -> u.text("你现在扮演盛唐著名的诗人{role}，接下来我们进行对话")
                    .param("role", role))
            .user(u -> u.text("我是一个现代诗歌爱好者，我的提问是：{msg}").params(Map.of("msg", msg)))
            .call().content();
}
```

<font style="color:rgb(31, 35, 40);">默认使用的是 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">{}</font>`<font style="color:rgb(31, 35, 40);"> 进行模板变量替换，当然如果你有诉求，可以用 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);"><></font>`<font style="color:rgb(31, 35, 40);"> 进行替换（如提示词中包含json时，</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">{}</font>`<font style="color:rgb(31, 35, 40);">的方式就不太适合），可以这样调整</font>

```java
@GetMapping("/ai/template")
public String template(@RequestParam(value = "role", defaultValue = "李白") String role,
                       @RequestParam(value = "msg", defaultValue = "你好") String msg) {
    return chatClient.prompt()
            .system(u -> u.text("你现在扮演盛唐著名的诗人{role}，接下来我们进行对话")
                    .param("role", role))
            .user(u -> u.text("我是一个现代诗歌爱好者，我的提问是：<msg>").params(Map.of("msg", msg)))
            .templateRenderer(StTemplateRenderer.builder().startDelimiterToken('<').endDelimiterToken('>').build())
            .call().content();
}
```

### <font style="color:rgb(31, 35, 40);">2. 流式结构化返回</font>
<font style="color:rgb(31, 35, 40);">对于流式场景，由于大模型是逐步返回的，没办法将完整的内容直接转换为目标对象，可以考虑使用下面的方式。</font>

```java
public class ChatController {

    @GetMapping(path = "/ai/fluxGenV2")
    public List<Poem> fluxGenV2(@RequestParam(value = "msg", defaultValue = "你好") String msg) {
        var converter = new BeanOutputConverter<>(new ParameterizedTypeReference<List<Poem>>() {
        });

        Flux<String> flux = chatClient.prompt()
                .system("你现在扮演盛唐著名的诗人李白，接下来我们进行对话")
                .user(u -> u.text("{msg}.\n{format}").param("msg", msg).param("format", converter.getFormat()))
                .stream().content();
        String content = flux.collectList().block().stream().collect(Collectors.joining());
        return converter.convert(content);
    }
}
```

### <font style="color:rgb(31, 35, 40);">3. 设置默认值</font>
<font style="color:rgb(31, 35, 40);">在创建</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClient</font>`<font style="color:rgb(31, 35, 40);">时，可以使用一些默认的系统消息、提示词设置（通过 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">defaultXxx</font>`<font style="color:rgb(31, 35, 40);"> 方式实现）。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">如下面给出了提供默认的消息提示词（支持带参数） 和默认的模型参数设置。说明：默认的配置，可以通过不带 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">default</font>`<font style="color:rgb(31, 35, 40);"> 前缀的方法进行覆盖。</font>

```java
@RestController
public class ChatController {

    private static final org.slf4j.Logger log = org.slf4j.LoggerFactory.getLogger(ChatController.class);

    private final ChatClient chatClient;

    private final ChatClient poemClient;

    public ChatController(ChatModel chatModel) {
        chatClient = ChatClient.builder(chatModel).build();

        poemClient = ChatClient.builder(chatModel)
                .defaultSystem("你现在扮演著名的诗人{role}，接下来我们进行对话")
                .defaultOptions(ChatOptions.builder().maxTokens(500).build())
                .build();
    }

    @GetMapping("/ai/poet")
    public Poem poetChat(String role, String msg) {
        return poemClient.prompt().system(sp -> sp.param("role", role))
                .user(msg)
                .call().entity(Poem.class);
    }


    record Poem(String title, String content) {
    }
}
```

### <font style="color:rgb(31, 35, 40);">4. 增强 Advisor</font>
`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 为 SpringAI 提供了更强大的拦截、修改和增强能力。这个思路基本和 AOP 类似，但是 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 允许在运行时动态修改方法调用。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">如果我们希望在用户消息的基础上追加或增强上下文数据时</font>

+ <font style="color:rgb(31, 35, 40);">可以通过 RAG 给大模型喂资料</font>
+ <font style="color:rgb(31, 35, 40);">也可以集成聊天历史，实现多轮对话</font>

<font style="color:rgb(31, 35, 40);">比如之前在介绍聊天上下文时，提到的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">MessageChatMemoryAdvisor</font>`<font style="color:rgb(31, 35, 40);">，可以用来实现多轮对话。</font>

```java
@Autowired
private ChatMemory chatMemory;

@GetMapping("/ai/historyChat")
public String historyChat(String msg) {
    return chatClient.prompt()
            .system("你现在扮演盛唐著名诗人李白，我们接下来开启对话")
            .user(msg)
            .advisors(MessageChatMemoryAdvisor.builder(chatMemory).build())
            .call().content();
}
```

<font style="color:rgb(31, 35, 40);">如果觉得默认的输出不合心意，也可以在创建时，指定传参、返回的打印方式，如</font>

```java
@GetMapping("/ai/historyChat")
public String historyChat(String msg) {
    return chatClient.prompt()
            .system("你现在扮演盛唐著名诗人李白，我们接下来开启对话")
            .user(msg)
            .advisors(new SimpleLoggerAdvisor(
                            req -> ("[request] " + req),
                            res -> ("[response] " + res),
                            0),
                    MessageChatMemoryAdvisor.builder(chatMemory).build())
            .call().content();
}
```

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">请注意，当传入多个 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">advisor</font>`<font style="color:rgb(31, 35, 40);"> 时，顺序很重要。每个 Advisor 都会以某种方式修改提示词或上下文，且一个 Advisor 的更改会传递给链中的下一个 Advisor。</font>
