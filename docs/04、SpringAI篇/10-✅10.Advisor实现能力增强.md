Spring AI 提供了 **Advisor API**，为在 Spring 应用里拦截、修改、增强 AI 交互提供了灵活而强大的能力支撑。  


它的核心优势可以概括为三点：

+ **模式封装**：把常见的生成式 AI 交互模式抽象成可复用的组件，让开发者不用重复造轮子；
+ **数据转换**：对和大语言模型（LLM）之间交互的数据进行拦截和转换，更好地适配不同场景；
+ **可移植性**：同一套逻辑可以跨模型、跨用例迁移，避免绑死在单一模型或具体实现上。

## <font style="color:rgb(31, 35, 40);">一、Advisor基础知识</font>
### <font style="color:rgb(31, 35, 40);">1. Advisor核心组件</font>
<font style="color:rgb(31, 35, 40);">因为大模型支持同步/流式两种方式，所以 SpringAI也提供同步调用的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">call</font>`<font style="color:rgb(31, 35, 40);"> 和流式调用的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">stream</font>`<font style="color:rgb(31, 35, 40);">。这两种方式，对应的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);">也不太一样。</font>

+ <font style="color:rgb(31, 35, 40);">流式：</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">StreamAdvisor</font>`<font style="color:rgb(31, 35, 40);"> </font><font style="color:rgb(31, 35, 40);">和</font><font style="color:rgb(31, 35, 40);"> </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">StreamAdvisorChain</font>`
+ <font style="color:rgb(31, 35, 40);">非流式：</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">CallAdvisor</font>`<font style="color:rgb(31, 35, 40);"> </font><font style="color:rgb(31, 35, 40);">和</font><font style="color:rgb(31, 35, 40);"> </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">CallAdvisorChain</font>`

<font style="color:rgb(31, 35, 40);">接口定义如下：</font>

```java
public interface StreamAdvisor extends Advisor {
    Flux<ChatClientResponse> adviseStream(ChatClientRequest chatClientRequest, StreamAdvisorChain streamAdvisorChain);
}

public interface CallAdvisor extends Advisor {
    ChatClientResponse adviseCall(ChatClientRequest chatClientRequest, CallAdvisorChain callAdvisorChain);
}
```

<font style="color:rgb(31, 35, 40);">其中</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClientRequest</font>`<font style="color:rgb(31, 35, 40);">表示请求参数，</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatClientResponse</font>`<font style="color:rgb(31, 35, 40);">表示响应结果。</font>

从继承关系图来看，Advisor 的核心方法是 **adviseCall** 和 **adviseStream**。这两个方法的职责可以理解为对一次完整的对话调用进行“拦截—增强—放行”的流程：

+ 在调用链开始，先检查 Prompt 数据是否完整、是否有未密封的风险；
+ 在必要时，对 Prompt 进行定制和增强（比如加上下文、拼接额外提示）；
+ 再调用 Advisor 链中的下一个节点，保证调用链继续向前；
+ 如果需要，可以选择性地阻断请求，不让它进入模型；
+ 对返回的聊天响应做分析或加工；
+ 如果发生错误，可以通过抛出异常的方式标识出来。  


除了核心逻辑，Advisor 还提供了两个辅助方法：

+ **getOrder()**：决定当前 Advisor 在调用链中的执行顺序，数值越小越先执行。
+ **getName()**：返回当前 Advisor 的唯一标识名称，方便在链路中区分和调试。

在 Spring AI 的交互过程中，**Advisor Chain** 就像是一条“拦截—增强—放行”的流水线。整个生命周期大致是这样的：

1. ****首先，用户的提示词会被封装成一个 ChatClientRequest 对象，同时还会创建一个空的 AdvisorContext，用来在链路中传递共享信息。
2. ****每个 Advisor 会依次对请求进行处理。它们既可以修改请求内容，也可以选择直接阻断请求，不再调用后续的 Advisor 或模型。如果选择阻断，那么该 Advisor 就要负责生成并填充响应结果。
3. ****如果请求一路没有被阻断，那么链条的最后一个 Advisor 会触发调用聊天模型。
4. ****模型的响应结果会返回给 Advisor Chain，并被封装成一个 AdvisedResponse 对象，其中包含共享的 AdvisorContext。
5. ****链上的每个 Advisor 会依次对响应进行处理或修改，可以做分析、增强、过滤等操作。
6. ****最终，通过提取 ChatCompletion 的内容，生成 ChatClientResponse，并返回给客户端。

### <font style="color:rgb(31, 35, 40);">3. 执行顺序</font>
<font style="color:rgb(31, 35, 40);">链中 Advisor 的执行顺序由 getOrder() 方法决定</font>

1. <font style="color:rgb(31, 35, 40);">数值越小，优先级越高</font>
2. `<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 链采用栈式结构运作，链首的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 优先处理请求，链尾的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 优先处理响应。</font>
3. <font style="color:rgb(31, 35, 40);">控制执行顺序，通过调整 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">order</font>`<font style="color:rgb(31, 35, 40);"> 值，来控制执行顺序，当多个</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);">的</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">order</font>`<font style="color:rgb(31, 35, 40);">值相同时，无法完全确认执行顺序</font>

## <font style="color:rgb(31, 35, 40);">二、自定义</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"></font>
<font style="color:rgb(31, 35, 40);">接下来我们通过实现一个自定义的 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 来实现大模型交互的耗时统计。</font>

### <font style="color:rgb(31, 35, 40);">1. 耗时统计Advisor</font>
<font style="color:rgb(31, 35, 40);">首先创建一个 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">CostAdvisor</font>`<font style="color:rgb(31, 35, 40);"> 分别实现 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">CallAdvisor</font>`<font style="color:rgb(31, 35, 40);"> 和 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">StreamAdvisor</font>`<font style="color:rgb(31, 35, 40);"> 接口，让它同时适用于同步/流式场景。</font>

<font style="color:rgb(31, 35, 40);">既然是耗时统计，那就设定这个 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);"> 优先级最高，在执行前后分别计时，用于获取耗时情况；并将相关信息写入到上下文中。</font>

```java
public class CostAdvisor implements CallAdvisor, StreamAdvisor {
    private static final Logger log = org.slf4j.LoggerFactory.getLogger(CostAdvisor.class);

    /**
     * 同步调用
     *
     * @param chatClientRequest
     * @param callAdvisorChain
     * @return
     */
    @Override
    public ChatClientResponse adviseCall(ChatClientRequest chatClientRequest, CallAdvisorChain callAdvisorChain) {
        long start = System.currentTimeMillis();
        chatClientRequest.context().put("start-time", start);

        // AdvisorChain 的调度执行 (有点类似 Filter 的链式执行 )
        ChatClientResponse response = callAdvisorChain.nextCall(chatClientRequest);

        long end = System.currentTimeMillis();
        long cost = end - start;
        response.context().put("end-time", end);
        response.context().put("cost-time", cost);
        log.info("Prompt call cost: {} ms", cost);
        return response;
    }

    /**
     * 流式调用
     *
     * @param chatClientRequest
     * @param streamAdvisorChain
     * @return
     */
    @Override
    public Flux<ChatClientResponse> adviseStream(ChatClientRequest chatClientRequest, StreamAdvisorChain streamAdvisorChain) {
        long start = System.currentTimeMillis();
        chatClientRequest.context().put("start-time", start);
        Flux<ChatClientResponse> response = streamAdvisorChain.nextStream(chatClientRequest);
        return new ChatClientMessageAggregator().aggregateChatClientResponse(response, (res) -> {
            long end = System.currentTimeMillis();
            long cost = end - start;
            // 添加耗时
            res.context().put("end-time", end);
            res.context().put("cost-time", cost);
            log.info("Prompt stream cost: {} ms", cost);
        });
    }

    @Override
    public String getName() {
        return "costAdvisor";
    }

    @Override
    public int getOrder() {
        // 指定最高优先级
        return Integer.MIN_VALUE;
    }
}
```

<font style="color:rgb(31, 35, 40);">同步的实现比较好理解，重点说一下流式调用中，耗时时间的打印情况，这里借助的是</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">MessageAggregator</font>`<font style="color:rgb(31, 35, 40);">工具类，用于将 Flux 响应聚合为 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">AdvisedResponse</font>`<font style="color:rgb(31, 35, 40);">。</font>

<font style="color:rgb(31, 35, 40);"></font>

<font style="color:rgb(31, 35, 40);">其次就是上下文的传递，可以通过 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">chatClientRequest.context().put("start-time", start);</font>`<font style="color:rgb(31, 35, 40);"> 和 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">response.context().put("end-time", end);</font>`<font style="color:rgb(31, 35, 40);"> 方式设置用于共享的上下文参数。</font>

### <font style="color:rgb(31, 35, 40);">2. 测试验证</font>

```java
@RestController
public class ChatController {

    private static final org.slf4j.Logger log = org.slf4j.LoggerFactory.getLogger(ChatController.class);

    private final ChatClient chatClient;

    public ChatController(ChatModel chatModel) {
        chatClient = ChatClient.builder(chatModel)
                .defaultSystem("你现在扮演盛唐的著名诗人李白，我们开始对话")
                .defaultAdvisors(new CostAdvisor())
                .build();
    }

    @GetMapping("/call")
    public String call(String msg) {
        ChatClientResponse response = chatClient.prompt(msg).call().chatClientResponse();
        log.info("上下文：{}", response.context());
        return response.chatResponse().getResult().getOutput().getText();
    }

    @GetMapping(path = "/stream", produces = "text/event-stream")
    public Flux<String> stream(String msg) {
        Flux<String> ans = chatClient.prompt(msg).stream().content();
        return ans;
    }
}
```

## <font style="color:rgb(31, 35, 40);">三、小结</font>
本篇文章主要介绍了 **Spring AI 如何通过 Advisor 来增强大模型的交互能力**，并且通过一个简单的 CostAdvisor 演示了如何实现自定义 Advisor。  


除了自定义实现，Spring AI 内置了许多常用的 Advisor，可以直接拿来用：

### **对话历史相关Advisor**
+ **MessageChatMemoryAdvisor，**检索对话记忆，并将其作为信息集合添加到提示中。这种方式可以保持对话历史的结构，不过需要注意，并不是所有模型都支持这种方法。
+ **PromptChatMemoryAdvisor，**把记忆内容直接整合到提示词的系统文本中，让模型在生成时带上上下文。
+ **VectorStoreChatMemoryAdvisor，**从向量存储库里检索相关记忆，并注入提示词的系统文本。这个 Advisor 特别适合在海量数据中高效搜索。

### **问答相关Advisor**
+ **QuestionAnswerAdvisor，**基于向量存储的检索增强生成（RAG）模式，实现问答功能。

### **内容安全Advisor**
+ **SafeGuardAdvisor，**基础的防护型 Advisor，用于阻止模型生成有害或不当内容。
