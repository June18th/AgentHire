### 一、基础技术栈
#### 1. 语言与框架
+ **JDK17**：作为长期支持版本(LTS)，JDK17带来了诸多新特性如密封类(sealed classes)、模式匹配增强等，为项目提供了稳定的运行时环境。
+ **Spring Boot 3**：基于Spring Framework 6构建，全面支持Jakarta EE 9+，带来了以下优势：
    - 原生镜像支持(AOT编译)
    - 改进的自动配置机制
    - 增强的监控和管理端点
    - 对JDK17特性的全面支持
+ **Spring AI 1.0**：Spring生态中新兴的AI集成框架，为项目提供了统一的大模型接入能力。
+ **LangGraph4J**:** **多智能体开发框架，定义（）循环图、使不同的组件（代理、工具或自定义逻辑）能够以有状态的方式进行交互

#### 2. 依赖与版本管理
+ **Maven**：采用传统的依赖管理工具，通过POM文件清晰定义项目结构和依赖关系。
+ **Git**：版本控制系统，配合合理的分支策略(如Git Flow)实现团队协作开发。

#### 3. 数据持久化方案
+ **数据库**：
    - 开发环境：H2 Database - 内存数据库，快速启动，适合开发和测试
    - 生产环境：MySQL - 成熟的关系型数据库，稳定可靠
+ **ORM**：
    - **Spring Data JPA**：简化数据访问层开发，提供：
        * 基于Repository的抽象
        * 方法名查询自动生成
        * 分页和排序支持
        * 与Hibernate的无缝集成
+ **数据库版本管理**：
    - **Liquibase**：数据库变更管理工具，支持：
        * 变更集(ChangeSet)的版本控制
        * 多环境数据库迁移
        * 回滚能力
        * 与Spring Boot的自动集成

#### 4. 缓存
+ **Guava Cache**：Google提供的本地缓存解决方案，特点包括：
    - 基于内存的高效缓存
    - 可配置的过期策略
    - 缓存淘汰机制
    - 统计信息收集

```java
        HttpSessionListener.super.sessionCreated(se);

    /**
     * session失效，在线人数统计数-1
     *
     * @param se
     */
    public void sessionDestroyed(HttpSessionEvent se) {
        HttpSessionListener.super.sessionDestroyed(se);
        SpringUtil.getBean(LoginService.class).incrOnlineUserCnt(-1);
    }
}
```

### 四、系统架构设计
#### 1. 事件驱动架构
+ **Spring事件机制**：实现业务解耦

```java
// 定义事件
@Getter
public class TaskChangeListener extends ApplicationEvent {

    private final Long taskId;
    private GatherTaskStateEnum state;

    public TaskChangeListener(Object source, Long taskId, GatherTaskStateEnum state) {
        super(source);
        this.taskId = taskId;
        this.state = state;
    }
}


// 发布任务新增事件，主动触发一次任务调度
SpringUtil.getContext().publishEvent(new TaskChangeListener(this, taskEntity.getId(), GatherTaskStateEnum.INIT));


@Async
@EventListener(TaskChangeListener.class)
public void taskListener(TaskChangeListener listener) {
    switch (listener.getState()) {
        // 触发任务执行
        case INIT -> scheduleToLoadTask();
        case SUCCEED, FAILED, PROCESSING -> {
            log.info("任务执行，可以给用户发送一个消息通知<任务编号:{} 状态:{}>", listener.getTaskId(), listener.getState());
        }
    }
}
```

```java
        return;
    }

    SCHEDULE_LOCK.set(true);
    try {
        //...
    } finally {
        SCHEDULE_LOCK.set(false);
    }
}
```

### 六、总结
求职派项目技术栈体现了现代Java开发的典型特征：

1. **现代化**：基于JDK17和Spring Boot3构建，拥抱最新技术
2. **AI集成**：通过Spring AI统一接入多种大模型，降低AI能力使用门槛
3. **微信生态**：完整对接微信公众号和支付，拓展用户触达渠道
4. **健壮架构**：结合事件驱动、任务调度、权限控制等机制构建稳定系统
5. **开发效率**：合理选择Hutool、FactExcel等工具库提升开发效率

总体来讲，技术栈主流又不复杂，业务聚焦，重心突出，代码工程结构简单、直接，自吹一句这个项目完全可以说是新手、初级的最佳启蒙项目
