# 01-✅求职派前置环境搭建

工程篇将给准备学习、改造求职派源码的小伙伴进行一系列基础性的科普。

本篇作为工程篇的第一篇，不展开项目启动细节，只先把运行求职派前需要的本机工具准备好。

后端开发需要 JDK 21 + Maven；如果要改前端页面，还需要 Node.js + pnpm。

MySQL 不是默认启动必须项，只有在你需要切换到 test 环境、验证 Liquibase 或模拟生产数据库时才需要安装。

接下来我将分别以后端、前端的视角，教大家如何搭建自己的前置开发环境。

## 1.后端运行环境

运行求职派后端，最基础的依赖是 JDK 21 + Maven；如果你只是想在本地快速启动项目，这两个环境准备好就够了。

如果你想将数据保存到 MySQL 中，那么还需要安装 MySQL 8+。

### a. jdk21安装

#### macOS 安装步骤

```bash
# 使用 Homebrew 安装 OpenJDK 21
brew install openjdk@21

# 创建符号链接以使系统可以找到 java 可执行文件
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk

# 添加到 ~/.zshrc 或 ~/.bash_profile 中
echo 'export PATH="$(brew --prefix)/opt/openjdk@21/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

验证安装

```bash
java -version
```

应显示类似以下输出：

```plain
openjdk version "21.0.x" 2024-xx-xx
OpenJDK Runtime Environment (build 21.0.x+x-xxxx)
OpenJDK 64-Bit Server VM (build 21.0.x+x-xxxx, mixed mode, sharing)
```

#### Windows 安装

Windows 的话，可以直接下载安装包，然后在 PATH 命令中设置路径就好了，我这里就不再截图，参考这个连接：

[手把手教你在Windows和macOS下安装JDK，菜逼也能掌握](https://javabetter.cn/overview/jdk-install-config.html)

说明：如果你的机器上已经安装了 JDK 8，怎么安装 JDK 21 呢？

1. 官网下载安装包：[https://www.oracle.com/java/technologies/downloads/#java21](https://www.oracle.com/java/technologies/downloads/#java21)
2. 一路下一步，开始安装
3. 无需配置环境变量
4. IDEA 启动，进入 Project Sturcture 设置，在 SDK 这里会自动识别到我们安装的版本（比如我的工作本上安装了 JDK 8/17/21，默认使用 JDK 8，手动切换选择 21）

### b. Maven 安装

求职派使用 Maven 作为项目依赖管理，当你从 GitCode 上将求职派下载下来，导入 IDEA，发现一片飙红，无法正常运行时，99% 的原因就是你的 Maven 有点问题；网上的安装方式非常多，下面给出基本的安装步骤。

**如果之前没有用过 Maven，看一下这个帖子：**

[终于把项目构建神器Maven捋清楚了~](https://javabetter.cn/maven/maven.html)

**记得把阿里云的镜像配置一下。**

Maven 的三端安装姿势差不多，唯一的区别在于环境变量的配置。

**下载地址：**[https://maven.apache.org/download.cgi](https://maven.apache.org/download.cgi)

选择 .tar.gz 或者 .zip 的包，本地选择一个位置进行解压。

#### Windows 环境配置

Windows 环境变量配置方式同上面的 JDK 方式，新增环境变量 MAVEN_HOME。

![](https://cdn.paicoding.com/paicoding/ce22c1b0f40d2d436c6497d5d274764e.png)

更新 Path 变量，添加 %MAVEN_HOME%\bin。

![](https://cdn.paicoding.com/paicoding/ecd4bf35b03ba1811d9ff2e7bf88a897.png)

确定之后，在终端中输入 **mvn --version** 验证。

![](https://cdn.paicoding.com/paicoding/5a40d7fcba0eba9eb866c63f147dbb3e.png)

#### Linux 环境配置

打开终端

```bash
vim ~/.bashrc

# 再文件的最后面，添加maven的解压目录
export MAVEN_HOME=/home/yihui/soft/apache-maven-3.6.3

export PATH=${MAVEN_HOME}/bin:${PATH}

# 保存并退出上面的文件编辑，然后让配置变更生效
source ~/.bashrc

# 验证
mvn --version
```

#### macOS 环境配置

同上 Linux 配置。

### c. MySQL 安装（非必须）

#### Windows 和 macOS 环境安装

这俩操作系统可以通过下面这篇内容查看安装方式。

[MySQL 的安装 from 二哥的 Java 进阶之路](https://javabetter.cn/mysql/install.html)

#### Linux 环境安装

**apt-install**

```bash
# 安装命令
sudo apt-get install mysql-server mysql-client -y

# 注意安装过程中的初始化密码设置（若没有，则可以通过下面的方法找到初始化密码）

grep "temporary password" /var/log/mysqld.log

## 输出如下
# A temporary password is generated for root@localhost: xxxx
```

**yum**

```bash
# 安装命令
yum install mysql mysql-server mysql-libs

# 初始密码查找

grep "temporary password" /var/log/mysqld.log

## 输出如下
# A temporary password is generated for root@localhost: xxxx
```

安装完毕之后，修改密码。

使用`set password`

**格式：**

```plain
mysql> set password for 用户名@localhost = password('新密码');
```

**例子：**

```plain
mysql> set password for root@localhost = password('123');
```

update 方式：

```plain
mysql> use mysql;

mysql> update user set password=password('123') where user='root' and host='localhost';

mysql> flush privileges;
```

添加用户：

```plain
alter user 'root'@'localhost' identified by 'test';
create user 'test'@'%' IDENTIFIED BY 'test';
```

授予权限：

```plain
# root 方式登录
grant all PRIVILEGES on test.* to 'yihui'@'%' IDENTIFIED by 'test';
flush privileges;
```

**数据库启动/关闭命令**

```bash
# 启动
sudo service mysql start
# 或 sudo service mysqld start

# 关闭
sudo service mysql stop

# 重启
sudo service mysql restart
```

## 2.前端运行环境

前端需要安装 Node.js，由于前端的不同版本之间的差异性较大，我贴一下我使用的前端环境。

- node: v18.19.0

包管理工具（三个我都安装了，求职派主要使用 pnpm 进行依赖安装）：

- npm: 10.2.3
- pnpm: 10.5.2
- yarn: 1.22.19

由于不同的前端工程对运行的 Node 版本要求不一致，我个人使用 nvm 进行管理切换，版本为 1.1.12。

### a. nvm 安装

**Windows 环境**

| 步骤           | 操作                                                                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| 1. 卸载旧 Node | 控制面板 → 程序 → 卸载 Node.js                                                                                                            |
| 2. 下载安装器  | [GitHub Releases](https://github.com/coreybutler/nvm-windows/releases) 下载 `nvm-setup.exe`                                               |
| 3. 安装        | 双击 → 全程 Next（**路径勿含中文或空格**）                                                                                                |
| 4. 验证        | `Win+R` → cmd → `nvm version` 能看到版本号即可                                                                                            |
| 5. 常用命令    | `nvm list available` 查看可装版本 `nvm install 20.15.1` `nvm use 20.15.1`                                                                 |
| 6. 换国内镜像  | 编辑 `%NVM_HOME%\settings.txt` 追加： `node_mirror: https://npmmirror.com/mirrors/node/` `npm_mirror: https://npmmirror.com/mirrors/npm/` |

**macOS 环境**

| **步骤**       | **操作**                                                                                                                         |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| 1. 安装依赖    | 确保已装 `git`、`curl`                                                                                                           |
| 2. 一键脚本    | 终端执行： `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh \| bash`                                    |
| 3. 配置 shell  | 把下面内容追加到 `~/.zshrc`（或 `~/.bashrc`）： `export NVM_DIR="$HOME/.nvm"` `[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"` |
| 4. 生效        | `source ~/.zshrc`                                                                                                                |
| 5. 验证        | `nvm --version`                                                                                                                  |
| 6. 安装 & 切换 | `nvm install --lts` `nvm use --lts`                                                                                              |

---

**Linux（Ubuntu / CentOS 通用）**

| **步骤**       | **操作**                                                                   |
| -------------- | -------------------------------------------------------------------------- |
| 1. 一键脚本    | `curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh \| bash` |
| 2. 配置 shell  | 同上，追加到 `~/.bashrc`（或 `~/.zshrc`）后 `source ~/.bashrc`            |
| 3. 验证 & 安装 | `nvm --version` `nvm install 18.20.3` `nvm use 18.20.3`                   |

nvm 安装之后，可以直接使用命令行的方式安装 Node.js，比如安装一灰灰的同款 Node.js。

```bash
nvm list available
nvm install 18.19.0
```

### b. pnpm 安装

Node 安装完毕之后，直接借助 npm 进行安装。

```bash
npm install -g pnpm@latest-10
```

国内镜像加速（可选）

```bash
pnpm config set registry https://registry.npmmirror.com
```
