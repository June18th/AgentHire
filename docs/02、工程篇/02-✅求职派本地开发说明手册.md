## 二、项目初始化与本地启动

### 1. 导入后端项目

导入到 IntelliJ IDEA 后是下面这个样子。

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617164133.png)

建议 Maven 的配置和下面的截图一致：Maven 主路径选择「使用 Maven 包装器」，重写用户设置文件不勾选，本地仓库不勾选，使用 `.mvn/maven.config` 中的设置勾选。

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617165528.png)

下面简单解释一下这几个选项的区别：

**① Maven 主路径：「使用 Maven 包装器」vs「已捆绑（Maven 3）」**

- **使用 Maven 包装器**：走项目根目录下的 `mvnw` 脚本来执行构建。好处是所有人用的 Maven 版本都由项目的 `.mvn/wrapper/maven-wrapper.properties` 统一控制，不管你本地装的是 Maven 3.6 还是 3.9，构建行为都一致。推荐选这个。
- **已捆绑（Maven 3）**：走 IDEA 自带的 Maven。如果你的 IDEA 版本比较新，内置的 Maven 版本通常也够用，但不同 IDEA 版本捆绑的 Maven 可能不一样，团队协作时容易出现“我这能跑你那跑不了”的情况。
- 还有一个「本地安装」选项，就是走你系统 `PATH` 里自己装的 Maven，版本完全取决于你个人环境。

**② 重写 — 用户设置文件：**

- **勾选**：可以指定一个自定义的 `settings.xml` 路径，IDEA 构建时会用你指定的这个文件，而不是默认的 `~/.m2/settings.xml`。适合你需要为这个项目单独配一套镜像源或私服地址的场景。
- **不勾选**：使用系统默认的 `~/.m2/settings.xml`。对于大多数人来说，这个文件里已经配好了阿里云镜像，不需要额外覆盖。

**③ 重写 — 本地仓库：**

- **勾选**：可以指定一个自定义目录作为这个项目的本地仓库，下载的 jar 包会放到你指定的位置。适合你想把某个项目的依赖和其他项目隔离开的场景。
- **不勾选**：所有项目共享默认的 `~/.m2/repository`，已经下载过的依赖可以直接复用，省磁盘也省下载时间。绝大多数情况下不需要改。

**④ 使用 .mvn/maven.config 中的设置：**

- **勾选**：IDEA 构建时会自动读取项目根目录下 `.mvn/maven.config` 里预设的参数（比如额外的仓库地址、编译选项等），和你在命令行跑 `./mvnw install` 的行为完全一致。**求职派项目必须勾选这个**，否则可能出现 langgraph4j 等依赖下载失败的问题。
- **不勾选**：IDEA 会忽略 `.mvn/maven.config`，只用 IDEA 自己的默认配置去构建，行为可能和命令行不一致。

导入完成后，建议先在终端执行一次全量构建，确保所有模块都编译通过：

```bash
./mvnw install -DskipTests
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617165703.png)

如果 `mvnw` 没有执行权限，先运行：

```bash
chmod +x mvnw
```

这一步会把各子模块安装到本地 Maven 仓库，后续 IDEA 编译和命令行启动都不容易遇到跨模块依赖找不到的问题。

### 2. 初始化环境变量文件

项目根目录下有 `.env.example`，先复制一份本地 `.env`：

```bash
cp .env.example .env
```

这份 `.env` 主要用于本地数据库名、MCP、支付等运行配置覆盖。

### 3. 拷贝 H2 初始化数据库

默认 dev 环境使用 H2。为了避免污染种子数据，建议复制一份本地开发库：

```bash
cp workspace/datas/jobclaw.mv.db workspace/datas/jobclaw-my.mv.db
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617171238.png)

然后在 `.env` 中修改数据库名称：

```bash
JOBCLAW_DATABASE_NAME=jobclaw-my
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617171302.png)

这样做的好处是：如果测试过程中数据混乱，可以直接删除 `jobclaw-my.mv.db` 并重新拷贝一份，不会影响原始种子数据。

### 4. 启动后端

方式一：IDEA 启动。

- 进入 `JobClawApplication` 主类入口
- 在 main 方法前面的启动标识按钮上点击启动

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617171523.png)

方式二：命令行启动。

```bash
./mvnw spring-boot:run -pl app
```

`-pl app` 指定只运行 app 模块，也就是 Spring Boot 主应用所在模块。

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617171622.png)

启动后直接访问：http://localhost:8087

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617171734.png)

:::warning
说明：

项目默认做了字典数据初始化，并集成了两个用户：一个普通用户、一个管理员。

虽然求职派是前后端分离项目，但后端 `app/src/main/resources/static/` 目录中已经放了打包好的前端静态资源。因此只启动后端，也能正常访问页面。

:::

### 1. 后端项目结构

后端使用的技术栈为：`Spring Boot 4.0.5 + Java 21 + Spring AI 2.0.0-M4 + LangGraph4J 1.6.0 + Spring JPA + JobRunr + JWT`

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618085109.png)

① 采用标准的后端 MVC 分层架构

- Controller：定义数据访问端点（即各种 HTTP 接口）
- Service：提供服务实现
- Dao：数据库访问

② 关键目录的功能划分如下所示：

```bash
├── agents/      # LangGraph4J 多智能体工作流（任务分类→数据采集→清洗→发布）
├── components/  # 通用组件
├── configs/     # 全局字典
├── constants/   # 常量定义
├── gather/      # AI大模型相关，任务采集
├── llm/         # LLM调用监控与管理
├── oc/          # 正式职位数据服务、MCP Server端点
├── openapi/     # 开放平台接口(主要是与技术派进行账号互通的实现)
├── user/        # 用户相关服务，包含登录、用户管理、会员充值等
├── util/        # 一些工具类
├── web/         # SpringMVC相关，提供controller入口，登录鉴权，全局管理等
```

③ 业务领域划分：

- agents：LangGraph4J 多智能体工作流（TaskClassify → TaskGather → DraftWasher → DraftPublish）
- configs：全局字典
- gather：AI 大模型 + 数据抓取解析任务（GatherAiAgent、OfferGatherService）
- llm：LLM 调用监控与管理
- oc：校招职位信息、MCP Server 端点
- user：用户 + 会员充值

④ 辅助工具层：

- openapi：开放平台接口(主要是与技术派进行账号互通的实现)
- components：通用组件，如异步调度、业务异常、全局上下文、雪花算法生成ID策略
- constants：常量定义，包括不同业务领域相关的枚举、常量等
- util：工具类集合，通常一个类就是一个工具，包含常见的 JsonUtil、IpUtil、DateUtil、SessionUtil 等

⑤ Web 端点访问层主要提供 Web 服务相关的合集。

## 五、本地开发配置

前面已经完成默认 H2 Dev 路径的初始化和启动。下面继续说明 MySQL、开发登录、支付等本地开发时常见的补充配置。

### 1. MySQL 数据库配置

对于已经安装了 MySQL 的小伙伴，强烈推荐使用 MySQL 作为数据存储层；H2 虽然简单，但稳定性和可靠性，以及功能的完备性相对来说要差一点；如果你本机已经安装了 MySQL，那么在本地开发时，请切换 `test` 开发环境。

#### 1. 切换 test 开发环境

![灰灰截图](https://cdn.paicoding.com/paicoding/7bb91c908e17639a5b8f85f73f03f4ed.png)

![二哥截图](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618084638.png)

#### 在 .env 中配置 MySQL 连接信息

test 环境的 `application-dal.yaml` 已经通过环境变量占位符引用 `.env`，无需直接修改 YAML 文件。只需在 `.env` 中设置以下配置即可：

```bash
DATABASE_HOST=localhost
DATABASE_PORT=3306
DATABASE_USERNAME=root
DATABASE_PASSWORD=你的MySQL密码
JOBCLAW_DATABASE_NAME=jobclaw
```

:::tip
H2 和 MySQL 的数据库名完全独立。

H2 用的是文件名（如 `jobclaw-my`），MySQL 用的是数据库名（如 `jobclaw`）。

切换 Maven Profile 即可切换数据源，`.env` 中的 `JOBCLAW_DATABASE_NAME` 在两个环境下分别指向不同的东西，按需修改即可。
:::

#### 启动项目，确定库表初始化情况

:::info
求职派与技术派一样，提供了自动创建库、表，基于 Liquibase 实现了数据库版本管理，因此你完全不需要手动创建表。

:::

第一次切到 test 环境时，建议先在项目根目录执行一次完整安装：

如果启动时看到类似下面的错误：

```text
Validation Failed:
1 changesets check sum
db/changelog/db.changelog-master.xml::260618_00000003::Codex was: ... but is now: ...
```

这不是 VSCode 终端或 Maven 命令的问题，而是当前 MySQL 库的 `databasechangelog` 表里记录过旧版 changelog 的 checksum，后来本地脚本内容又改了，Liquibase 为了避免已执行脚本被静默篡改而终止启动。

如果本地库里没有需要保留的数据，可以直接重建本地开发库：

```bash
mysql -h localhost -P 3306 -u root -p -e "DROP DATABASE IF EXISTS jobclaw; CREATE DATABASE jobclaw DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
```

如果要保留本地数据，只刷新这条变更集的 checksum：

```bash
mysql -h localhost -P 3306 -u root -p jobclaw -e "UPDATE databasechangelog SET md5sum = NULL WHERE id='260618_00000003' AND author='Codex';"
```

然后重新启动：

```bash
./mvnw -Ptest -pl app spring-boot:run
```

如果要在命令行确认库表数量，可以执行：

```bash
mysql -h localhost -P 3306 -u root -p -e "SELECT table_name FROM information_schema.tables WHERE table_schema='jobclaw' ORDER BY table_name;"
```

### 2. 初始化账号管理

初始的数据中，默认提供了两个用户，一个管理员，一个普通用户。

为了解决本地开发时，微信公众号的回调无法直接打到本地应用导致无法登录的情况（当然可以做内网穿透），求职派提供了开发环境快速登录能力。

#### 本地开发登录（无需配置）

**本地开发时不需要任何额外配置**，前端会自动检测当前访问地址是否为 `localhost`（或 `127.0.0.1`）。

如果是，登录弹窗会自动切换为“本地登录”模式，直接显示「普通用户登录」和「管理员登录」两个按钮，点击即可一键登录。

操作步骤：

1. 启动后端，访问 `http://localhost:8087`
2. 点击右上角「登录」按钮
3. 弹窗标题显示“本地登录”，直接选择身份即可

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618093900.png)

:::tip
原理：前端 `AppLayout.tsx` 中判断 `localLoginEnabled = localHost || isDevEnv`，本地访问时 `localHost` 为 true，自动启用。

登录时调用后端 `/api/wx/dev/login` 接口，传入 `type=user` 或 `type=admin` 区分身份。
:::

#### 非 localhost 环境启用开发环境登录

如果你的开发环境不是通过 localhost 访问的（比如用了 IP 或域名），需要手动开启：

1. 先用管理员身份登录（首次可通过 localhost 登录后再切换访问方式）
2. 进入后台管理：点击右上角头像 → 选择「管理后台」→ 左侧菜单选择「字典管理」
3. 找到 `全局-站点配置`，将 `env` 对应的 `value` 设置为 `dev`

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618104403.png)

### 1. 用户认证与权限管理

1. 支持关注微信公众号，发送验证码登录的方式。
2. 基于 JWT 的用户登录鉴权，实现了无状态的身份验证。

### 2. 基于大模型的求职信息采集 Agent

1. 提交采集任务后，系统会在任务表中插入一条记录，然后后台异步执行采集任务，不影响前端用户的操作。

![](https://cdn.paicoding.com/paicoding/47b129acb7cf6a0791267812e1f52dbd.png)

### 6. 多渠道智能体系统

求职派的核心亮点，基于 `core` 模块实现 Multi-Agent 消息管道：

- **IM 渠道接入**：微信（ClawBot）、钉钉、飞书，每个渠道独立模块，通过 `Channel.adaptToReceive()` 统一消息格式
- **消息路由**：`MsgRouter` 统一处理消息流转 — 身份采集 → 系统命令 → 意图识别 → 智能体路由
- **意图分类**：`IntentClassifier` 关键词 + LLM 复合分类，将用户消息路由到正确的业务智能体
- **业务智能体**：身份采集（identity-collector）、职位获取（job-fetch）、职位推荐（job-recommend）

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-f0d4c4bf980d246f5e4c463a23a8d79a.png)

### 7. MCP Server

求职派集成了 Spring AI 的 MCP Server 能力，对外暴露职位查询等工具，可被其他 AI 应用（如 Claude Desktop）直接调用：

- 协议配置在 `application.yml` 中的 `spring.ai.mcp.server` 下，默认开启
- 支持 SSE 和 Streamable HTTP 两种传输方式
- 注意区分：`.env` 中的 `AGENT_MCP_CLIENT_ENABLED` 控制的是 MCP **客户端**（求职派连接外部 MCP 服务器），与 MCP Server 对外暴露工具是两回事

### 8. 用户身份三层体系

求职派为每个用户维护三层画像，存储在 `workspace/users/{userId}/` 下：

- **Soul**（`soul/`）：个性特征、职业志向（AI 提取）
- **Identity**（`user/`）：技能、教育、工作经历
- **Info**（`info/`）：基础人口统计信息

由 `IIdentityAgent.triggerToCollectIdentity()` 在 `MsgRouter` 中自动触发，当用户画像不完整时主动采集。

### 9. LangGraph4J 多智能体工作流

`app/agents/` 下基于 LangGraph4J 实现了数据处理的多阶段工作流：

```
TaskClassify → TaskGather → DraftWasher → DraftPublish
```

用于校招信息从提交到发布的全流程自动化。

### 7. 任务池

用户提交的任务附件，如 CSV/Excel/图片，是保存到本地的，数据库中存储的是引用地址而不是文件内容。

本地文件的读写由 `LocalStorageHelper` 实现。

![](https://cdn.paicoding.com/paicoding/fe0a70f53bc070f3a0010e8119656df5.png)

本地文件存储路径通过 `.env` 配置：

```bash
JOBCLAW_IMG_ABS_TMP_PATH=/tmp/storage/       # 本地文件存储根路径
JOBCLAW_IMG_WEB_IMG_PATH=/oc/img/             # Web 访问路径
JOBCLAW_IMG_CDN_HOST=http://localhost:8087    # CDN / 本地访问域名
```

对应 `application.yml` 中 `jobclaw.img` 下的配置项。

![](https://cdn.paicoding.com/paicoding/19916d5062e3e7c9f1991c2fe02e7123.png)

任务池调度采用两种方式：

1. **Spring @Scheduled：**`OfferGatherService` 中通过 `@Scheduled(cron = "0 0/5 * * * ?")` 定时扫描待处理任务
2. **JobRunr：**`core` 模块中基于 JobRunr 实现更灵活的后台任务调度（Dashboard 在 8099 端口），适用于智能体的异步任务管理
