## 1.后端项目结构
后端使用的技术栈为：`SpringBoot 3.5.3 + JDK17 + SpringAI 1.0.0 + Spring JPA + JWT`


采用标准的后端MVC分层架构

+ Controller： 定义数据访问端点（即各种http接口）
+ Service:  提供服务实现
+ Dao: 数据库访问

```bash
├── agents/      # 求职派智能体
├── components/  # 通用组件
├── configs/     # 全局字典
├── constants/   # 常量定义
├── gather/      # ai大模型相关，任务采集
├── oc/          # 正式职位数据服务
├── openapi/     # 开放平台接口(主要是与技术派进行账号互通的实现)
├── user/        # 用户相关服务，包含登录、用户管理、会员充值等
├── util/        # 一些工具类
├── web/         # SpringMVC相关，提供controller入口，登录鉴权，全局管理等
```

业务领域划分：

+  agents: 求职派智能体的实现
+ configs: 全局字典
+ gather: ai大模型 + 数据抓取解析任务
+ oc:  校招职位信息
+ user:  用户 + 会员充值

辅助工具层：

+ openapi：开放平台接口(主要是与技术派进行账号互通的实现)
+ components：通用组建，如异步调度、业务异常、全局上下文、雪花算法生成ID策略
+ constants:  常量定义，包括不同业务领域相关的枚举、常量等
+ util：工具类集合，通常一个类就是一个工具，包含常见的JsonUtil, IpUtil, DateUtil, SessionUtil等

WEB端点访问层：

+ web:  提供web服务相关的合集

# 四、本地开发配置
## 1.AI配置
我们默认使用的是智谱清言作为底层大模型进行数据提取，因此在本地开发时，你需要先去申请一个APIKey [✅求职派大模型-智普清言接入](https://www.yuque.com/itwanger/yyt72l/bkb31aukyil47osx)

:::danger
说明：本地开发时，强烈推荐通过启动时命令行参数传入配置 apiKey，而不是直接修改配置文件；可以有效避免密钥泄露

:::

## 2.H2数据库配置
使用dev开发时，默认使用的是 h2dabase，对应的配置信息如下

+ url中指定了数据库的地址为当前项目根目录的datas下面
+ 数据库名在 `application-oc.yml`中定义了，默认为 `JobClaw`


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753755395404-1fd4a881-a690-4097-970a-a4293cbcde6c.png)


首次启动项目时，数据库中已经默认提供了两个用户和全局字典表，从我个人的开发习惯来说，不建议大家直接使用这个库：

1. 保障原始库的纯粹性
2. 当源头项目中的这个库结构变更之后，容易处理代码冲突

推荐的做法是：

1. 从 `workspace/datas/jobclaw.mv.db` 拷贝一个` workspace/datas/jobclaw-my.mv.db` 文件


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753755788526-651dd381-de10-4fb5-a600-da31a2d937f4.png)

2. 修改启动命令行参数，设置 `--jobclaw.database.name=jobclaw-my`，**注意和前面的智谱 key 有个空格隔开哦**。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753755812287-ed50bad3-6336-4abe-bf90-bc17ae42c196.png)

## 3.MySql数据库配置
对于已经安装了 MySQL 的小伙伴，强烈推荐使用MySql作为数据存储 层；h2虽然简单，但稳定性和可靠性，以及功能的完备性相对来说要差一点；如果你本机已经安装了 mysql，那么在本地开发时，请切换 `test` 开发环境

1. 切换test开发环境


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753756151579-d16a0e60-93f1-4e64-a168-221ddcd6231a.png)

2. 更新mysql数据库链接信息
+ 进入resources-env/test/application-dal.yml
+ 修改 url，username，password，根据你真实的数据库链接信息进行替换


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753756207519-fc495255-2be1-4786-ae4a-3022fcbd4377.png)

3. 启动项目，确定库表初始化情况

:::info
求职派与技术派一样，提供了自动创建库、表，基于liquibase实现了数据库版本管理，因此你完全不需要创建库、表，直接启动项目就行了

:::

启动完毕之后，刷新你的数据库连接工具，默认的库名为 `oc`，当项目正常启动之后就可以看到下面的输出日志，并可以看到数据库中一个库


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753756414887-e9810819-d9ec-404b-a58a-fb1b60e263b9.png)


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753756452004-d9d1d416-17da-4517-a192-ffac932d125a.png)


:::danger
如果你的数据库不支持Liquibase，那也没关系，更新配置 `spring.liquibase.enabled = false` ，求职派实现了一套专属的库表初始化逻辑，从而实现自动创建库表、sql初始化

:::


假如你已经切换到了 test 的 Maven 配置，但实际执行的时候仍然是通过 dev，那说明 Maven 的缓存被污染了。重新执行一下下面的命令应该就好了。


```java

```

# 五、五大功能模块
## 1.用户认证与权限管理
1. 支持关注微信公众号，发送验证码登录的方式，便捷又安全，降低了用户注册登录的门槛。
2. 基于 jwt（JSON Web Token）的用户登录鉴权，实现了无状态的身份验证。同时，自定义的权限管理模块，能精准控制不同用户的操作权限，保障系统数据安全。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753758640134-7482dc20-3e38-48c1-b367-1e1b60298228.png)

## 2.基于大模型的校招信息采集Agent
借助 SpringAI 实现大模型的数据采集，这是求职派的一大技术亮点。

1. 提交采集任务后，系统会在任务表中插入一条记录，然后后台异步执行采集任务，不影响前端用户的操作。
2. 支持多种数据输入方式，包括文本、图片、csv 文件、excel 文件、http 网页、http 链接等，能全面覆盖各类校招信息的来源，满足多样化的采集需求。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753758693255-4de32eda-0a6c-41f8-b9dc-bf5e0a33456d.png)

## 3.异步AI调度任务机制
我们设计了一个任务池的数据模型，将用户提交的采集任务先放到任务池，然后进行异步的任务调度执行，从而减少大模型响应慢的问题

采集数据后，支持数据清洗、同步到正式的oc库，采用采集数据与正式数据分离的思想，避免污染业务库。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753758722020-e4bc5ea7-c923-4261-8c11-b30fe52a807e.png)
