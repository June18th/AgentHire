### 1. 后端项目结构

后端使用的技术栈为：`Spring Boot 4.0.5 + Java 21 + Spring AI 2.0.0-M4 + LangGraph4J 1.6.0 + Spring JPA + JobRunr + JWT`

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618085109.png)

① 采用标准的后端 MVC 分层架构

- Controller：定义数据访问端点（即各种 HTTP 接口）
- Service：提供服务实现
- Dao：数据库访问

② 关键目录的功能划分如下所示：

```bash
├── agents/      # LangGraph4J 多智能体工作流（任务分类→数据采集→清洗→发布）
├── components/  # 通用组件
├── configs/     # 全局字典
├── constants/   # 常量定义
├── gather/      # AI大模型相关，任务采集
├── llm/         # LLM调用监控与管理
├── oc/          # 正式职位数据服务、MCP Server端点
├── openapi/     # 开放平台接口(主要是与技术派进行账号互通的实现)
├── user/        # 用户相关服务，包含登录、用户管理、会员充值等
├── util/        # 一些工具类
├── web/         # SpringMVC相关，提供controller入口，登录鉴权，全局管理等
```

③ 业务领域划分：

- agents：LangGraph4J 多智能体工作流（TaskClassify → TaskGather → DraftWasher → DraftPublish）
- configs：全局字典
- gather：AI 大模型 + 数据抓取解析任务（GatherAiAgent、OfferGatherService）
- llm：LLM 调用监控与管理
- oc：校招职位信息、MCP Server 端点
- user：用户 + 会员充值

④ 辅助工具层：

- openapi：开放平台接口(主要是与技术派进行账号互通的实现)
- components：通用组件，如异步调度、业务异常、全局上下文、雪花算法生成ID策略
- constants：常量定义，包括不同业务领域相关的枚举、常量等
- util：工具类集合，通常一个类就是一个工具，包含常见的 JsonUtil、IpUtil、DateUtil、SessionUtil 等

⑤ Web 端点访问层主要提供 Web 服务相关的合集。

## 四、本地开发配置

`.env` 环境变量、智谱 API Key、H2 数据库拷贝等基础配置已在上一篇 [✅求职派本地开发环境搭建指南](https://paicoding.com/jobclaw-dev-environment-setup) 中详细介绍，这里不再重复。

本篇聚焦上一篇未涉及的进阶配置。

### 1. MySQL 数据库配置

对于已经安装了 MySQL 的小伙伴，强烈推荐使用 MySQL 作为数据存储层；H2 虽然简单，但稳定性和可靠性，以及功能的完备性相对来说要差一点；如果你本机已经安装了 MySQL，那么在本地开发时，请切换 `test` 开发环境。

#### 1. 切换 test 开发环境

![灰灰截图](https://cdn.paicoding.com/paicoding/7bb91c908e17639a5b8f85f73f03f4ed.png)

![二哥截图](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618084638.png)

#### 在 .env 中配置 MySQL 连接信息

test 环境的 `application-dal.yaml` 已经通过环境变量占位符引用 `.env`，无需直接修改 YAML 文件。只需在 `.env` 中设置以下配置即可：

```bash
DATABASE_HOST=localhost
DATABASE_PORT=3306
DATABASE_USERNAME=root
DATABASE_PASSWORD=你的MySQL密码
JOBCLAW_DATABASE_NAME=jobclaw
```

:::tip
H2 和 MySQL 的数据库名完全独立。

H2 用的是文件名（如 `jobclaw-my`），MySQL 用的是数据库名（如 `jobclaw`）。

切换 Maven Profile 即可切换数据源，`.env` 中的 `JOBCLAW_DATABASE_NAME` 在两个环境下分别指向不同的东西，按需修改即可。
:::

#### 启动项目，确定库表初始化情况

:::info
求职派与技术派一样，提供了自动创建库、表，基于 Liquibase 实现了数据库版本管理，因此你完全不需要手动创建表。

:::

第一次切到 test 环境时，建议先在项目根目录执行一次完整安装：

```bash
./mvnw -U -Ptest -DskipTests clean install
```

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618092523.png)

### 2. 初始化账号管理

初始的数据中，默认提供了两个用户，一个管理员，一个普通用户。

为了解决本地开发时，微信公众号的回调无法直接打到本地应用导致无法登录的情况（当然可以做内网穿透），求职派提供了开发环境快速登录能力。

#### 本地开发登录（无需配置）

**本地开发时不需要任何额外配置**，前端会自动检测当前访问地址是否为 `localhost`（或 `127.0.0.1`）。

如果是，登录弹窗会自动切换为“本地登录”模式，直接显示「普通用户登录」和「管理员登录」两个按钮，点击即可一键登录。

操作步骤：

1. 启动后端，访问 `http://localhost:8087`
2. 点击右上角「登录」按钮
3. 弹窗标题显示“本地登录”，直接选择身份即可

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618093900.png)

:::tip
原理：前端 `AppLayout.tsx` 中判断 `localLoginEnabled = localHost || isDevEnv`，本地访问时 `localHost` 为 true，自动启用。

登录时调用后端 `/api/wx/dev/login` 接口，传入 `type=user` 或 `type=admin` 区分身份。
:::

#### 非 localhost 环境启用开发环境登录

如果你的开发环境不是通过 localhost 访问的（比如用了 IP 或域名），需要手动开启：

1. 先用管理员身份登录（首次可通过 localhost 登录后再切换访问方式）
2. 进入后台管理：点击右上角头像 → 选择「管理后台」→ 左侧菜单选择「字典管理」
3. 找到 `全局-站点配置`，将 `env` 对应的 `value` 设置为 `dev`

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-20260618104403.png)

### 1. 用户认证与权限管理

1. 支持关注微信公众号，发送验证码登录的方式。
2. 基于 JWT 的用户登录鉴权，实现了无状态的身份验证。

![](https://cdn.paicoding.com/paicoding/3d5e3ad664ef555e4e1b5fcff3bb3857.png)

### 2. 基于大模型的求职信息采集 Agent

1. 提交采集任务后，系统会在任务表中插入一条记录，然后后台异步执行采集任务，不影响前端用户的操作。
2. 支持多种数据输入方式，包括文本、图片、CSV 文件、Excel 文件、HTTP 网页、HTTP 链接等，能全面覆盖各类求职信息的来源，满足多样化的采集需求。

![](https://cdn.paicoding.com/paicoding/bb073c5fc4e392e53139ff65b9704358.png)

### 3. 异步 AI 调度任务机制

我们设计了一个任务池的数据模型，将用户提交的采集任务先放到任务池，然后进行异步的任务调度执行。

采集数据后，支持数据清洗、同步到正式的 oc 库，采用采集数据与正式数据分离的思想，避免污染业务库。

![](https://cdn.paicoding.com/paicoding/37830f519dcf2c195ed96896dd13fcd7.png)

### 6. 多渠道智能体系统

求职派的核心亮点，基于 `core` 模块实现 Multi-Agent 消息管道：

- **IM 渠道接入**：微信（ClawBot）、钉钉、飞书，每个渠道独立模块，通过 `Channel.adaptToReceive()` 统一消息格式
- **消息路由**：`MsgRouter` 统一处理消息流转 — 身份采集 → 系统命令 → 意图识别 → 智能体路由
- **意图分类**：`IntentClassifier` 关键词 + LLM 复合分类，将用户消息路由到正确的业务智能体
- **业务智能体**：身份采集（identity-collector）、职位获取（job-fetch）、职位推荐（job-recommend）

![](https://cdn.paicoding.com/stutymore/02-✅求职派本地开发说明手册-f0d4c4bf980d246f5e4c463a23a8d79a.png)

### 7. MCP Server

求职派集成了 Spring AI 的 MCP Server 能力，对外暴露职位查询等工具，可被其他 AI 应用（如 Claude Desktop）直接调用：

- 协议配置在 `application.yml` 中的 `spring.ai.mcp.server` 下，默认开启
- 支持 SSE 和 Streamable HTTP 两种传输方式
- 注意区分：`.env` 中的 `AGENT_MCP_CLIENT_ENABLED` 控制的是 MCP **客户端**（求职派连接外部 MCP 服务器），与 MCP Server 对外暴露工具是两回事

### 8. 用户身份三层体系

求职派为每个用户维护三层画像，存储在 `workspace/users/{userId}/` 下：

- **Soul**（`soul/`）：个性特征、职业志向（AI 提取）
- **Identity**（`user/`）：技能、教育、工作经历
- **Info**（`info/`）：基础人口统计信息

由 `IIdentityAgent.triggerToCollectIdentity()` 在 `MsgRouter` 中自动触发，当用户画像不完整时主动采集。

### 9. LangGraph4J 多智能体工作流

`app/agents/` 下基于 LangGraph4J 实现了数据处理的多阶段工作流：

```
TaskClassify → TaskGather → DraftWasher → DraftPublish
```

用于校招信息从提交到发布的全流程自动化。

### 7. 任务池

用户提交的任务附件，如 CSV/Excel/图片，是保存到本地的，数据库中存储的是引用地址而不是文件内容。

本地文件的读写由 `LocalStorageHelper` 实现。

![](https://cdn.paicoding.com/paicoding/fe0a70f53bc070f3a0010e8119656df5.png)

本地文件存储路径通过 `.env` 配置：

```bash
JOBCLAW_IMG_ABS_TMP_PATH=/tmp/storage/       # 本地文件存储根路径
JOBCLAW_IMG_WEB_IMG_PATH=/oc/img/             # Web 访问路径
JOBCLAW_IMG_CDN_HOST=http://localhost:8087    # CDN / 本地访问域名
```

对应 `application.yml` 中 `jobclaw.img` 下的配置项。

![](https://cdn.paicoding.com/paicoding/19916d5062e3e7c9f1991c2fe02e7123.png)

任务池调度采用两种方式：

1. **Spring @Scheduled：**`OfferGatherService` 中通过 `@Scheduled(cron = "0 0/5 * * * ?")` 定时扫描待处理任务
2. **JobRunr：**`core` 模块中基于 JobRunr 实现更灵活的后台任务调度（Dashboard 在 8099 端口），适用于智能体的异步任务管理

![](https://cdn.paicoding.com/paicoding/d1be042a94a5806a49ff4c48b01cc0a0.png)
