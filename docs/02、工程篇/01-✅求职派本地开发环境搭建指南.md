### 1.后端运行环境

运行求职派，最基础的依赖只有两个：JDK 21 + Maven；如果你只是想在本地进行开发，那么搭建上面两个环境足以；如果你想将数据保存到 MySQL 中，那么你可能还需要安装 MySQL 8+。

#### a. jdk21安装

##### macOS 安装步骤

```bash
# 使用 Homebrew 安装 OpenJDK 21
brew install openjdk@21

# 创建符号链接以使系统可以找到 java 可执行文件
sudo ln -sfn $(brew --prefix)/opt/openjdk@21/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-21.jdk

# 添加到 ~/.zshrc 或 ~/.bash_profile 中
echo 'export PATH="$(brew --prefix)/opt/openjdk@21/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

验证安装

```bash
java -version
```

应显示类似以下输出：

```plain
openjdk version "21.0.x" 2024-xx-xx
OpenJDK Runtime Environment (build 21.0.x+x-xxxx)
OpenJDK 64-Bit Server VM (build 21.0.x+x-xxxx, mixed mode, sharing)
```

##### Linux 环境安装

**apt-install**

```bash

# 注意安装过程中的初始化密码设置（若没有，则可以通过下面的方法找到初始化密码）
```

**yum**

```bash

## 输出如下

```

安装完毕之后，修改密码。

使用`set password`

**格式：**

```plain
mysql> set password for 用户名@localhost = password('新密码');
```

**例子：**

```plain
mysql> set password for root@localhost = password('123');
```

update 方式：

```plain
mysql> use mysql;

mysql> update user set password=password('123') where user='root' and host='localhost';

mysql> flush privileges;
```

添加用户：

```plain
alter user 'root'@'localhost' identified by 'test';
create user 'test'@'%' IDENTIFIED BY 'test';
```

授予权限：

```plain
# root 方式登录
grant all PRIVILEGES on test.* to 'yihui'@'%' IDENTIFIED by 'test';
flush privileges;
```

**数据库启动/关闭命令**

```bash
# 启动
sudo service mysql start
# 或 sudo service mysqld start

# 关闭
sudo service mysql stop

# 重启
sudo service mysql restart
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617165528.png)

下面简单解释一下这几个选项的区别：

**① Maven 主路径：「使用 Maven 包装器」vs「已捆绑（Maven 3）」**

- **使用 Maven 包装器**：走项目根目录下的 `mvnw` 脚本来执行构建。好处是所有人用的 Maven 版本都由项目的 `.mvn/wrapper/maven-wrapper.properties` 统一控制，不管你本地装的是 Maven 3.6 还是 3.9，构建行为都一致。推荐选这个。
- **已捆绑（Maven 3）**：走 IDEA 自带的 Maven。如果你的 IDEA 版本比较新，内置的 Maven 版本通常也够用，但不同 IDEA 版本捆绑的 Maven 可能不一样，团队协作时容易出现“我这能跑你那跑不了”的情况。
- 还有一个「本地安装」选项，就是走你系统 `PATH` 里自己装的 Maven，版本完全取决于你个人环境。

**② 重写 — 用户设置文件：**

- **勾选**：可以指定一个自定义的 `settings.xml` 路径，IDEA 构建时会用你指定的这个文件，而不是默认的 `~/.m2/settings.xml`。适合你需要为这个项目单独配一套镜像源或私服地址的场景。
- **不勾选**：使用系统默认的 `~/.m2/settings.xml`。对于大多数人来说，这个文件里已经配好了阿里云镜像，不需要额外覆盖。

**③ 重写 — 本地仓库：**

- **勾选**：可以指定一个自定义目录作为这个项目的本地仓库，下载的 jar 包会放到你指定的位置。适合你想把某个项目的依赖和其他项目隔离开的场景。
- **不勾选**：所有项目共享默认的 `~/.m2/repository`，已经下载过的依赖可以直接复用，省磁盘也省下载时间。绝大多数情况下不需要改。

**④ 使用 .mvn/maven.config 中的设置：**

- **勾选**：IDEA 构建时会自动读取项目根目录下 `.mvn/maven.config` 里预设的参数（比如额外的仓库地址、编译选项等），和你在命令行跑 `./mvnw install` 的行为完全一致。**求职派项目必须勾选这个**，否则可能出现 langgraph4j 等依赖下载失败的问题。
- **不勾选**：IDEA 会忽略 `.mvn/maven.config`，只用 IDEA 自己的默认配置去构建，行为可能和命令行不一致。

导入完成后，**建议先在终端执行一次全量构建**，确保所有模块（core、channels、providers、plugins、agents）都编译通过：

```bash
./mvnw install -DskipTests
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617165703.png)

> 如果 `mvnw` 没有执行权限，先运行：`chmod +x mvnw`

这一步会把各子模块安装到本地 Maven 仓库，后续 IDEA 编译和命令行启动都不会再遇到跨模块依赖找不到的问题。

:::warning
说明：

大家启动的项目，已经默认做了字典数据初始化，并集成了两个用户（一个普通用户、一个管理员）。

可能有小伙伴会问：不是说前后端分离吗？为何后端项目启动了，前端页面可以直接访问呢？

这里的「前后端分离」指的是：开发时前后端是独立的板块，采用不同的技术栈；可以各自独立打包、独立部署；前端通过 REST 接口与后端进行数据交互。

为啥后端启动后前端页面就能直接访问呢？

因为我们的后台应用本身也是基于 Tomcat 运行的完整 Web 应用，除了提供 REST 接口之外，还支持静态资源访问。

为了降低大家的本地开发难度，我们直接将前端打包好的静态资源放在了后端的资源目录下（`app/src/main/resources/static/`），这样一个 Web 应用就能同时提供前后端能力。

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617172109.png)

:::

### 第四步：初始化前端项目（可选）

**说明**：后端项目的 `app/src/main/resources/static/` 目录中已经包含了打包好的前端静态资源，所以**只启动后端就可以正常访问页面**。

下面的前端步骤仅在你需要**修改前端代码**时才需要执行。如果你只关注后端开发，可以直接跳过。

前端开发工具较多，你甚至可以直接在 IDEA 中进行开发；当然我个人更建议使用专业的前端开发工具（比如 Visual Studio、TRAE、VS Code）。

![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753452278837-32d910e0-fc8a-4927-bbad-2f9c72ebdb90.png)

注意，这里需要切换到 ui-react 目录下。

![](https://cdn.nlark.com/yuque/0/2025/png/12564477/1756783008909-c3aef714-9ea2-44b3-a41d-4fa8ee69fc3d.png)

### 第五步：安装前端依赖

打开终端，执行命令

```bash
pnpm install
```

不想手敲代码的话，记得打开 readme，里面有命令可以直接复制粘贴，贼贴心的哦。

![](https://cdn.nlark.com/yuque/0/2025/png/12564477/1756783119563-25e17dab-6434-4839-9ef0-c69744c0a9f8.png)

### 第六步：启动前端

同样是在终端内，执行命令

```bash
pnpm dev
```

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617172454.png)

启动成功之后，在控制台会看到上面的输出（直接戳链接，记得按住 Ctrl 键就可以直接默认浏览器打开），它默认使用 3000 端口，和前面后端访问的页面一致。

![](https://cdn.paicoding.com/stutymore/01-✅求职派本地开发环境搭建指南-20260617172514.png)

### 第七步：前端开发页面发布

当我们修改了前端页面之后，我们希望将改动同步到后端的静态资源目录下时，在终端内，执行下面的命令即可

```bash
pnpm run deploy
```

![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753452626040-98e6e07b-e05b-425d-bea0-6a4692feb802.png)

没有任何异常时，输出情况如上图，即表示成功将页面发布到了后端，当后端重启（如果热加载，则无需重启）之后，就会看到前端的改动了。

![](https://cdn.nlark.com/yuque/0/2025/png/12564477/1756783398885-802bea9f-b7bb-44e0-b87f-911af834331a.png)
