# 13-✅AI大模型应用开发：集成阿里云百炼实现公众号自动发文

在 [✅应用篇：MCP自动发布校招信息公众号推文](https://www.yuque.com/itwanger/yyt72l/upul837zhzoon8oc) 中，我们演示了如何借助 **MCP Server** 来实现求职派的自动 / 定时发布微信公众号功能。整体流程虽然跑通了，但在实际测试中我们发现——使用免费的智谱大模型时，表现并不稳定，尤其在调用复杂工具或处理长上下文内容时，**MCP Server 的回调失败率偏高**。


为了让系统真正具备生产可用的稳定性，我们决定尝试另一条路线：**集成阿里的千问大模型（Qwen）**，并通过它来完成微信公众号的自动发布。


这一篇文章会详细介绍整个过程：从接入千问 API、到模型工具调用配置，再到最终的公众号自动发布落地，带你看看如何让“智能发布”这件事，从实验变成可上线的功能。

## 一、阿里云百炼集成
### 1.添加依赖
在 Spring AI 的官方文档和 SDK 里，暂时还没有提供对阿里大模型的官方 Starter 支持。不过这并不是什么大问题——我们完全不需要自己去手搓大模型的交互逻辑。


毕竟阿里是**Java 届的老大哥**，自家的生态早就考虑到这一点，推出了 **Spring AI Alibaba**，专门为国内开发者做了深度整合。


而求职派的整体架构本身就是基于 Spring AI 运行的，所以直接使用阿里提供的 Starter，几乎可以无缝衔接，不管是模型调用、工具注册，还是配置管理，都和原生 Spring AI 的风格保持一致。


一句话总结：**不造轮子，直接上车。**我们要做的，只是接入千问模型，然后让它在求职派的智能体体系中跑起来。直接添加关键依赖。

```xml
        <!--  阿里的大模型      -->
        <dependency>
            <groupId>com.alibaba.cloud.ai</groupId>
            <artifactId>spring-ai-alibaba-starter-dashscope</artifactId>
            <version>1.0.0.3</version>
        </dependency>

```

+ 注意：multi-model 这个是求职派的自定义配置，主要用来放图片理解的模型
+ api-key:  百炼平台申请的api-key

### 3.集成百炼模型
在枚举中，新增百炼模型 GatherModelEnum。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1756117764368-a1a27b79-0e2a-49e1-b52e-0a250a5a8da2.png)

同步这一步也别忘了——我们需要在**全局字典**里，把这个新接入的模型配置上去。这样前端在读取配置的时候，才能正确识别到我们新增的模型选项，不然用户在页面上根本选不到。


简单来说，就是：**后端加配置，前端才能知道它存在。**


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1756173137939-37ff5f76-b8b3-497a-9970-199464bde165.png)

实现抽象类：`AbsOcChatModelApi`。记得指定图片理解的模型、对话模型等信息。

```java
@Component
public class AliBaiLianChatModel extends AbsOcChatModelApi {
    private final ChatModel chatModel;

    private final ChatClient chatClient;
    private ChatClient imgClient;

    /**
     * 多模态的图片理解模型
     */
    @Value("${spring.ai.dashscope.multi-model:qwen-omni-turbo}")
    private String sigImgModel;

    public AliBaiLianChatModel(DashScopeChatModel chatModel, List<McpAsyncClient> mcpClients) {
        this.chatModel = chatModel;

        chatClient = ChatClient.builder(chatModel)
                .defaultSystem(GATHER_SYSTEM_PROMPT)
                .defaultAdvisors(new SimpleLoggerAdvisor())
                // 将MCP Client 注册为工具
                .defaultToolCallbacks(new AsyncMcpToolCallbackProvider(mcpClients))
                .build();
    }

    @PostConstruct
    public void postInitImgClint() {
        // 图片理解
        imgClient = ChatClient.builder(chatModel)
                .defaultSystem(GATHER_SYSTEM_PROMPT)
                .defaultOptions(ChatOptions.builder()
                        .model(sigImgModel)
                        .stopSequences(Collections.emptyList()).build())
                .defaultAdvisors(new SimpleLoggerAdvisor())
                .build();
    }

    @Override
    public GatherModelEnum modelEnum() {
        return GatherModelEnum.ALI_BAILIAN;
    }


    @Override
    public ChatClient chatClient() {
        return chatClient;
    }

    @Override
    public ChatClient imgClient() {
        return imgClient;
    }

    @Override
    public ChatModel chatModel() {
        return chatModel;
    }

    @Override
    public String chatModelName() {
        return chatModel.getDefaultOptions().getModel();
    }

    @Override
    public String imgModelName() {
        return sigImgModel;
    }
}
```

## 三、小结
本文主要介绍了**阿里百炼大模型的接入过程**，并通过它演示了如何实现微信公众号的自动发布功能。


不得不说，不同大模型在真实场景下的表现确实有明显差异，这也是做多模型接入时非常有意思的部分。


下面简单讲一下，**求职派集成新大模型的标准流程**。无论是接入阿里、智谱、讯飞，还是未来其他厂商的模型，整体步骤都大同小异：

****

**①、申请大模型 API Key，**首先在对应的大模型平台上开通访问权限，拿到 API Key，这是调用模型的基础凭证。

****

**②、集成模型调用客户端**

+ 如果该模型提供了官方的 starter（比如阿里的百炼），那最简单，只需要在 pom.xml 中添加依赖即可。
+ 如果模型没有提供 starter（比如讯飞的星火），那就需要自定义实现 ChatModel，封装好 API 调用逻辑。

****

**③、扩展模型枚举类 GatherModelEnum，**在系统中注册新的模型类型，便于后续统一管理。记得同步更新全局字典，否则前端是感知不到这个新模型的。

****

**④、实现自定义抽象类 AbsOcChatModelApi，**这是核心的一步。关键点在于初始化两个 ChatClient：

+ 一个用于文本对话；
+ 一个用于多模态场景（比如图文理解）。


这样一套下来，新模型就能顺利接入求职派体系，参与任务分类、数据提取、内容生成等完整流程。
