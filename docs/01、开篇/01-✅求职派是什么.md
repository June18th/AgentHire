# 01-✅求职派是什么

### 一、一句话总结
用 Java 技术栈 + Spring AI，把 OpenClaw 风格的 Channel、Router、Agent、Provider、Plugin 分层落到真实求职场景里，做成一个可私有化部署、可扩展的多 Agent 实战项目。

> 当前版本说明：求职派已经从早期“岗位信息采集、解析、展示平台”，演进为类似 OpenClaw 架构的多 Agent Runtime。下面的视频和功能介绍保留了早期 V1 的演示口径，理解当前代码时请优先阅读 [求职派 OpenClaw 式多 Agent 架构总览](../03、架构篇/00-✅求职派OpenClaw式多Agent架构总览.md)。

### 三、核心功能模块亮点
#### 1.主流技术栈
求职派采用前后端分离的架构设计，在技术选型上紧跟主流且注重高效实用。

| 维度 | 选型 | 说明 |
| --- | --- | --- |
| 后端框架 | Spring Boot 4 + Java 21 | 使用 Java 后端熟悉的技术栈进行开发 |
| AI 接入 | Spring AI 2.0.0-M4 | 实现大模型的交互管理 |
| 数据层 | JPA + H2/MySQL | H2 快速本地跑，MySQL 一键切生产 |
| 前端 | React + Next.js 15 | SSR/SSG 兼顾 SEO 与体验 |


#### 2.五大功能模块

![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753610853418-6495af41-fdd0-4dee-9148-bfcef10d931b.png)

求职派系统架构

基于上面基于业务视角的系统架构图，我们将整个求职派划分了五大功能模块：

1. 实现登录、鉴权的 **用户认证与权限管理** 模块
2. 基于 Spring AI 实现的大模型能力应用层 **岗位采集/推荐等多 Agent 能力**
3. 支撑SpringAI采集任务的 **移步AI调度任务 **模块
4. 以微信支付为基座实现的 **会员充值 **模块
5. 统一维护管理全局字典、业务配置的 **全局业务字典管理 **模块


接下来将分别进行说明

##### （一）用户认证与权限管理
1. 支持关注微信公众号，发送验证码登录的方式，便捷又安全，降低了用户注册登录的门槛。
2. 基于 jwt（JSON Web Token）的用户登录鉴权，实现了无状态的身份验证，提高了系统的可扩展性。同时，自定义的权限管理模块，能精准控制不同用户的操作权限，保障系统数据安全。

> 说明：为了简化本地开发，我们提供了mock登录的功能
>


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753353279962-3bcec4c7-1453-4b9e-8810-d29c21290900.png)

公众号扫码登录

##### （二）基于大模型的校招信息采集Agent
借助 SpringAI 实现大模型的数据采集，这是求职派的一大技术亮点。

1. 提交采集任务后，系统会在任务表中插入一条记录，然后后台异步执行采集任务，不影响前端用户的操作，提升了系统的响应速度和用户体验。
2. 支持多种数据输入方式，包括文本、图片、csv 文件、excel 文件、http 网页、http 链接等，能全面覆盖各类校招信息的来源，满足多样化的采集需求。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753353505060-2fdc697a-8c0e-49de-b6f8-c572aedff24b.png)

多种ai采集任务方式


##### （三）异步AI调度任务机制
我们设计了一个任务池的数据模型，将用户提交的采集任务先下方的任务池，然后进行异步的任务调度执行，从而减少大模型响应慢的问题

采集数据后，支持数据清洗、同步到正式的oc库，采用采集数据与正式数据分离的思想，避免污染业务库。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753353527320-b0770ca9-2ff7-443e-9d87-d9123fd23075.png)

ai采集任务执行情况

##### （四）会员充值功能
集成微信支付实现会员充值，为项目的商业化运营提供了支持，开发者可基于此拓展更多增值服务。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753353816804-4112aca6-177a-4604-b95b-49ced92caffa.png)

微信支付，会员充值

##### （五）全局字典管理
统一维护全栈的配置信息，便于开发者对系统进行配置管理和维护，提高了系统的灵活性和可扩展性。


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753353581725-0964778b-b4a5-48ad-a3c0-e864443dd7ad.png)

### 四、项目源码结构速览
上面介绍了求职派是一个前后端分离的项目，重点说了一下五大功能模块，既有jwt/微信扫码登录鉴权、又有支付会员充值，还有大模型的数据采集，甚至是还塞进去了一个任务池，这个项目会不会很复杂、学习成本很高呢？

很负责任的说，完全不用有这个顾虑，这个项目的出发点非常单纯，让大家零成本的体验大模型开发的乐趣，天天在网上刷到的AiAgent，MCP我们也可以插一手


为了核心目的，在设计这个项目之处，就摒弃了一些大家在耳熟能详的中间件（比如redis、es、mq、微服务、oss、mybatis，甚至是mysql也可以不依赖）, 从前面的五分钟预览，相信大家也有一个直观的感受，一个jdk我就能跑这么多功能的应用了吗？这里面到底有什么魔法呢？


后续的文章，我讲逐渐给大家展开，这篇文章，就带大家来一个整体的预览

#### 1.工程结构说明
前后端项目，放在了同一个仓库中，其中 backend/ 下面的是后端工程源码； react-ui/ 下面的是前端工程源码


![](https://cdn.nlark.com/yuque/0/2025/png/35158118/1753355097038-ede848bf-3957-4a7f-a005-7f7c28586d00.png)

```bash
├── backend/      # 后端工程源码
├── data/          # 这里放的是 h2 数据库
├── docs/      	  # 这里放的是项目相关的文档说明
├── ui-react/          # 前端工程源码

```


#### 2.后端项目结构
采用标准的后端MVC分层架构

+ Controller： 定义数据访问端点（即各种http接口）
+ Service:  提供服务实现
+ Dao: 数据库访问

```bash
├── components/  # 通用组件
├── configs/     # 全局字典
├── constants/   # 常量定义
├── gather/      # ai大模型相关，任务采集
├── oc/          # 正式职位数据服务
├── user/        # 用户相关服务，包含登录、用户管理、会员充值等
├── util/        # 一些工具类
├── web/         # SpringMVC相关，提供controller入口，登录鉴权，全局管理等
```

### 五、小结


**最后给大家小结一下，求职派到底是什么**

求职派就是个 **SpringBoot3 打底，用 SpringAI 调用大模型当“信息提取器”** 的开源项目。专门对付各种格式的校招信息，自动把它们变成规整的结构化数据展示出来。包含了前后端分离、用户权限、异步任务、支付集成这些实用模块，代码可以直接参考，尤其想玩 **SpringAI 做智能信息提取**的兄弟，千万别错过！
