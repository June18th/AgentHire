项目名：校招派 Java 后端开发 2025 年 9 月-2025 年 10 月

项目介绍：

技术栈：SpringAI、H2、MySQL、MCP

核心职责：

1. 借助 SpringAI 实现大模型的数据采集 Agent，可自动化采集文本、图片、CVS、Excel、HTTP 网页和链接等多种来源，满足多样化的采集需求。
2. **Spring AI 实现的 MCP Server 与 MCP Client 的上下文信息是保存在 JVM 内存中的（通过 ConcurrentHashMap 维护）。**当后端服务重启后，这些会话数据会被清空，导致 MCP Server 无法识别来自旧 Client 的请求，从而无法正常响应。我通过重写 <font style="color:#000000;background-color:#c7edcc;">WebMvcSseServerTransportProvider，并</font>将会话信息持久化到 MySQL 来彻底解决这个问题。
3. 自定义 MCP Server 供 LLM 访问，支持工具调用、SSE 通信，也支持在 TRAE 中进行配置和访问。
4. 通过 SpringAI 集成 MCP Client 以调用外部服务，如  mcp-selenium，从而完成在 Chrome 浏览器中自动采集目标数据。
5. <font style="color:#0e0e0e;">基于 Spring AI 与 LangGraph4J 搭建多智能体系统，实现任务自动分类、采集、清洗、发布全流程；通过 AgentState 状态共享与异步消息优化，提升数据处理吞吐量 1.5 倍，显著降低人工清洗成本。</font>
6. <font style="color:rgb(0, 0, 0);">基于 Spring AI Advisor 机制与 ChatMemory，将历史对话与外部知识动态注入 Prompt，实现上下轮语境连贯与知识增强问答。</font>
7. 通过实现`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">CallAdvisor</font>`<font style="color:rgb(31, 35, 40);"> 和 </font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">StreamAdvisor</font>`<font style="color:rgb(31, 35, 40);"> 接口实现</font>自定义的`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Advisor</font>`<font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">来对大模型交互进行耗时统计。</font>
8. 借助 SpringAI 进行提示词 Prompt 的微调、<font style="color:rgb(31, 35, 40);">Converter 的结构化输出、</font><font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">ChatMemory</font><font style="color:rgb(31, 35, 40);"> 的多轮对话等</font>，使模型的输出结果更符合我们的预期。
9. 设计自定义的任务池数据模型，将用户提交的采集任务先放到任务池，再通过 RocketMQ 异步执行任务调度，从而减少 LLM 响应慢的问题。
10. 针对大模型一次无法完整返回结果的情况，我们借助 SpringAI 的 ChatMemo 实现了多轮对话的能力。对于多轮次的返回结果我们也做了定制化的 JSON 解析和拼接。
11. 对于 Excel 和 CSV 文件，我们通过预拆分+ FastExcel +循环调用大模型的方式来确保输入输出信息不被截断。
12. 封装自定义的 ChatModel，以接入无法使用 starter 直接导入依赖的模型，如讯飞星火的 <font style="color:rgb(31, 35, 40);">Spark Lite。</font>
13. 负责设计并实现一个基于用户个人偏好的订阅推荐系统，通过用户的自然语言描述，调用大模型生成结构化的SQL查询，从数据库中获取匹配的岗位信息并推荐给用户。
14. 利用智谱大模型+<font style="color:rgb(31, 35, 40);">文颜 </font>MCP Server，实现了自动化发布校园招聘信息到微信公众号。
15. 通过门面设计模式+策略模式+模板涉及模式接入多模型，以实现快速切换。
16. 基于 HTTP Authorization 进行 MCP Server 的请求鉴权
17. 通过工具调用 Function Call，给大模型扩展可以抓取 HTTP 网页内容的能力（Jsoup+Hutool），然后通过 @Tool 注解将其注册为 Function Tool。
18. 通过 SpringAI 的 Starter 接入智谱清言的 `<font style="color:rgb(19, 18, 18);">GLM-Z1-Flash</font>`<font style="color:rgb(19, 18, 18);"> 和 </font><font style="color:rgb(19, 18, 18);background-color:rgb(245, 247, 250);">GLM-4V-Flash</font>，以实现截图的文本识别。
19. 接入讯飞星火的 SparkLite、阿里百炼的通义千问-Plus 模型进行基本的文本处理，并支持通过策略模式一键切换。
20. 数据采集后，支持数据清洗，支持采集数据与正式数据分离的思想，避免污染业务库。
21. 实现基于 AOP 的自定义权限校验注解 @Permission，可在方法和类上进行接口的权限校验。
22. 利用 Maven 的 profile 实现多环境配置，包括测试、开发和生产等。
23. 启动项目时可通过命令行参数传入智谱清言大模型的 API Key，避免在配置文件中泄露密钥。
24. 默认使用 H2 数据库，但可以通过 Maven 一键切换至 MySQL，并且通过 Liquibase 实现了数据库的版本管理，支持自动创建库表，省去手工配置的麻烦。
25. 通过配置全局字典中 env 字段来控制弹窗的 mock 按钮是否显示，从而解决微信公众号无法回调本地开发环境的烦恼。
26. 接入微信商户 API V3 支付，以支持会员充值。
27. 支持微信公众号扫码登录，降低用户的注册登录门槛；基于 JWT 实现无状态的用户身份认证。同时，自定义的权限管理模块还可以精准控制管理员、付费会员和普通用户的数据隔离。
28. 通过全局字典，维护预定义的字段信息，如公司类型、招聘对象等，避免修改的时候需要同步更新前后端代码，并且支持热更新。
29. 通过在 IntelliJ IDEA 的 <font style="color:rgb(31, 35, 40);background-color:rgba(129, 139, 152, 0.12);">Program arguments</font> 中/或者环境变量中注入大模型的 API-KEY 来避免密钥的泄露；直接写死在配置文件中容易误提交到 Git 记录。
