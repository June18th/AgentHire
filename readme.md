# ai-oc

## 后端工程

后端相关工程放置在 [app](app/) 目录下

### 技术栈

jdk17 + SpringBoot3.5.3 + H2/MySql + SpringJPA + SpringAI + LangGraph4J

### 启动

本地开发时，数据库指向h2，直接启动即可；如果需要体验大模型的数据抓取录入，需要修改启动参数

1. 到智谱清言申请账号，注册一个API Key
    - api申请地址: [智谱清言API Key](https://open.bigmodel.cn/usercenter/proj-mgmt/apikeys)

2. 传入大模型ApiKey
   a. 命令行传参方式
    - 编辑启动命令
    - 点击 Modify options, 在菜单栏中，开启 `Program arguments`
    - 添加命令行参数 `--zhipuai-api-key=xxx`

   ![命令行配置.webp](docs/imgs/01-1.webp)

   b. 直接修改配置参数
    - 打开文件： [application-ai.yml](app/src/main/resources-env/dev/application-ai.yml)
    - 修改参数: `api-key:`

3. 入口类，直接启动

说明：

- dev 环境：使用h2数据库, 对应的数据库文件为：[app/src/main/resources/ai-oc.mv.db](app/src/main/resources/ai-oc.mv.db)
- test/prod 环境：使用MySql数据库

## 前端工程

前端相关工程放置在 [ui-react](ui-react/) 目录下

### 技术栈

react + next.js + tailwindcss

推荐nodejs=18.x, next.js=15+

### 启动

```bash
# 安装依赖
pnpm install

# 本地开发
pnpm dev

# 打包
pnpm build 

# 发布到SpringBoot的static目录下
pnpm run deploy
```

开发过程中的挑战：

- [ ] 工具太多，全部注册还是按需选择性注册？ 一次性注册还是动态注册？
  - 工具太多有什么问题？
  - 工具执行失败如何处理？工具调用失忆或者参数缺失错误
  - 在模型的交互层，如何实现按需加载工具（中间添加一层工具搜索层？）
  - 敏感工具执行前的授权如何实现？
  - 一个业务操作需要多个工具协作，如何实现（比如下单场景，需要收集用户收货地址、下单商品信息、送货方式等，假设每个收集方式都是一个独立的工具，需要这几个收集工具执行完毕之后，才能正确下单，那么这些工具如何协作呢？）
- [ ] 意图识别解决什么场景，如何实现？
  - 为什么要做意图识别？ 因为JobClaw的底层，提供了多个不同的业务Agent（如岗位信息收集、岗位信息推送、岗位信息推荐等），需要根据用户的意图，自动路由到合适的业务Agent
  - 在哪一层实现意图识别呢？ 在IM消息接收层，通过模型识别用户的意图，然后自动路由到合适的业务Agent
  - 如何实现呢? 关键词匹配 / 模型识别 / 预定义激活关键词
- [ ] 多轮对话的状态一致性如何保障？
  - 如上面意图识别，用户现在在之前的对话中已经选中了 xxAgent，如果每一轮对话都会先经过意图识别模块，那么如何确保下一次的对话中还是与 xxAgent进行对话呢？如果被路由到其他的Agent的怎么办？
  - 状态一致性的保障，需要保障的就是用户会话的连续性，不会突然被路由到其他业务Agent、从而导致响应幻觉的出现
  - 需要注意的是，用户的对话过程不可控，虽然之前的对话触发了xxAgent，可能我下一次的对话就是需要触发yyAgent，这种咋处理呢？（比如先触发岗位收集Agent，然后让它基于最新新的信息给一下推荐--> 这时候应该触发推荐Agent）
  - 如何保障多步骤流程可以按照预期的执行，不会因为中间穿插其他的打断而导致幻觉或者流程崩溃的出现呢？
- [ ] 模型可靠性
  - 多模型的切换：如何实现不同模型的无缝切换
  - 模型推理和决策的可靠性如何衡量，是无脑接受大模型的一切返回，还是设计一套衡量标准？
- [ ] 多Agent交互、协作
  - 多Agent之间如何进行信息交换、相互发现调用等
- [ ] 上下文窗口有限，但是会话的交互无限，如何解决这两者之间的矛盾冲突
  - 因为窗口有限，我们无法将所有的会话内容全部丢到上下文，那么如何处理这种情况呢?
  - 压缩对话，如何确保关键的信息没有丢失呢？固化下来的内容是准确的呢？
  - 多轮会话的场景中需要注意什么？何时触发对话历史压缩、丢弃无用的对话呢
- [ ] 不同用户/相同用户的不同的会话隔离怎么做？ 同一个用户、不同会话的用户共性特性如何共享？
- [ ] 记忆工程有什么用？为什么需要？如何实现？
- [ ] 复杂的任务，如何进行拆解、分布实现？执行过程中出现异常，如何处理？
  - Pan-and-Execute 如何实现的？其指导原理是什么？
- [ ] 如何实现让Agent永不停息的工作，而不是一问一答的被动式触发方式呢？
- [ ] 如何实现越用越智能、越用越懂你？
  - 如何设计一套拥有自主进化、自学习的Agent呢？
- [ ] 安全与监控应该做些什么？如何做？
  - 提示注入：攻击者通过恶意输入篡改 Agent 行为。 ‌应对‌：隔离系统指令与用户输入，对输出编码处理
  - 数据隐私：敏感信息直接传给大模型是否会导致泄露？ 应对：数据最小化、传输脱敏、严格访问控制
  - 越权操作与工具滥用
  - 合规审计
- [ ] 如何利用大模型的能力，给现有的服务应用进行赋能？
